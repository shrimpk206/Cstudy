# 02. 자주 나오는 에러 5종

vibe coding을 하면서 만나는 에러는 거의 다 이 5종 안에 들어갑니다. 각각의 **전형적 원인**을 외워두면 진단이 빨라집니다.

---

## 1) TypeError — "이 값을 그렇게 쓸 수 없다"

```
TypeError: Cannot read properties of undefined (reading 'name')
TypeError: someFunction is not a function
TypeError: Cannot read properties of null (reading 'map')
```

**의미:** 어떤 값에 대해 그 타입에 맞지 않는 동작을 했다.

**자주 일어나는 상황:**
- 객체가 `undefined`인데 속성을 꺼내려 함 → `user.name`인데 user가 undefined
- 함수가 아닌데 호출함 → `data.filter()`인데 data가 함수 없는 객체
- 배열인 줄 알았는데 객체 → `users.map()` 했는데 users가 객체

**고치는 법:**
```javascript
// ❌
const name = user.name

// ✅ 옵셔널 체이닝
const name = user?.name

// ✅ 또는 명시적 검사
const name = user ? user.name : "이름없음"
```

**AI에게 던질 때:**
> "TypeError: Cannot read properties of undefined (reading 'X'). users.js:12에서 났는데, 그 줄 코드는 `user.X`. user가 undefined일 가능성이 있나? 그렇다면 안전 처리 추가해줘."

---

## 2) ReferenceError — "그런 이름 못 봤는데?"

```
ReferenceError: foo is not defined
```

**의미:** 그 이름의 변수/함수를 찾을 수 없다.

**자주 일어나는 상황:**
- 오타: `consle.log` (console 아니고 consle)
- import 누락: 다른 파일의 함수를 import 안 했음
- 스코프 문제: `if {}` 안에서 만든 변수를 밖에서 씀
- 호이스팅 문제: `let`/`const`를 선언 전에 사용

**고치는 법:**
- 이름 오타 확인
- 파일 맨 위 `import` 확인
- 변수 선언 위치 점검

**AI에게:**
> "ReferenceError: foo is not defined — main.js:5에서. foo는 utils.js에 있는데, import가 빠진 듯. 추가해줘."

---

## 3) SyntaxError — "문법이 안 맞아서 못 읽어"

```
SyntaxError: Unexpected token '}'
SyntaxError: Unexpected end of input
SyntaxError: Invalid or unexpected token
```

**의미:** 코드를 읽다가 문법이 깨진 곳 발견. **실행도 못 들어감.**

**자주 일어나는 상황:**
- 괄호/중괄호/대괄호 짝 안 맞음
- 따옴표 안 닫힘
- 쉼표 위치 이상
- 영어 코드인데 한글 따옴표("")가 섞임

**고치는 법:**
- 에디터의 자동 들여쓰기 점검 (들여쓰기가 어긋나면 짝이 안 맞은 것)
- 에디터에 깔린 ESLint/Prettier 활용
- 에러가 "Unexpected end of input"이면 끝까지 안 닫힘

**AI에게:**
> "SyntaxError가 났는데 어디가 문제인지 모르겠어. 전체 파일 코드 붙여줄게: [코드]. 문법 오류 찾아줘."

---

## 4) Module Not Found — "그런 라이브러리/파일 없어"

```
Error: Cannot find module 'axios'
Module not found: Can't resolve './utils'
ModuleNotFoundError: No module named 'requests'   ← Python
```

**의미:** import한 모듈을 찾을 수 없다.

**자주 일어나는 상황:**
- 라이브러리를 설치 안 함 → `npm install axios` 빠짐
- 경로 오타 → `./utlis` (utils 아니고 utlis)
- 대소문자 틀림 → `./Button` 인데 파일은 `button.js` (Mac/Linux는 구분)
- 확장자 빠짐 → 환경에 따라 `./utils.js`로 명시 필요

**고치는 법:**
- 외부 라이브러리면: `npm install xxx` 또는 `pip install xxx`
- 본인 파일이면: 경로 / 대소문자 / 확장자 점검

**AI에게:**
> "Cannot find module './Button' 에러. components 폴더에 Button.tsx 파일 있어. import 경로가 틀렸나?"

---

## 5) Network / API 에러

```
Failed to fetch
NetworkError when attempting to fetch resource
Request failed with status code 401
CORS policy: No 'Access-Control-Allow-Origin' header
```

**의미:** 외부 호출이 실패했다.

**자주 일어나는 상황:**
- **CORS 에러**: 브라우저에서 다른 도메인으로 직접 호출했고 서버가 허용 안 함
- **401 Unauthorized**: API 키/토큰 빠짐
- **404**: URL 오타
- **Failed to fetch**: 인터넷 끊김, 또는 서버 다운, 또는 URL이 실재 안 함
- **CERT_HAS_EXPIRED**: 인증서 만료 (드뭄)

**CORS 에러 특별 안내:**
이건 **백엔드 문제거나, 프록시를 둬야 해결됩니다**. 프론트엔드 코드만 고쳐서는 안 풀립니다.
- 본인 백엔드면 → CORS 헤더 추가
- 남의 API면 → 백엔드(서버) 거쳐서 호출

**AI에게:**
> "API 호출에 CORS 에러. 브라우저에서 직접 https://other-api.com을 호출하고 있어. 우리 백엔드(Express)에서 프록시 엔드포인트 만들어서 거기로 보내게 바꿔줘."

---

## 빠른 진단표

| 에러 첫 단어 | 1순위 의심 | 어떻게 고치나 |
|---|---|---|
| TypeError | 값이 undefined/null이거나 기대한 타입 아님 | 옵셔널 체이닝, 타입 검사 |
| ReferenceError | 오타, import 누락, 스코프 | 이름 확인, import 추가 |
| SyntaxError | 괄호/따옴표 짝 | 에디터 들여쓰기로 추적 |
| Cannot find module | 설치 안 함 또는 경로 틀림 | install 또는 경로 수정 |
| Failed to fetch / CORS | 네트워크/인증/도메인 | 환경변수, 프록시, 백엔드 처리 |

각 에러의 종류와 가장 의심해야 할 원인을 한 줄로 답하세요.

1. `TypeError: result.map is not a function`
2. `ReferenceError: useStat is not defined`
3. `Module not found: Error: Can't resolve 'axios'`

<details>
<summary>정답</summary>

1. TypeError. result가 배열이 아닌데 `.map()`을 호출. 응답 형식 점검 (혹시 객체로 옴? 비어 있음?).
2. ReferenceError. 거의 확실히 `useState` 오타.
3. Module not found. `npm install axios` 안 했음.
