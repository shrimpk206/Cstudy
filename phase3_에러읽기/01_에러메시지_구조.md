# 01. 에러 메시지의 구조 — 스택 트레이스 읽기

## 핵심 한 줄

에러는 무서운 게 아니라 **정보**입니다. 어디서, 무엇이, 왜 잘못됐는지 거의 다 적혀 있습니다. 읽는 법만 알면 절반은 해결됩니다.

---

## 1) 전형적인 에러 메시지

```
TypeError: Cannot read properties of undefined (reading 'name')
    at getUser (/Users/me/app/src/users.js:12:18)
    at processData (/Users/me/app/src/main.js:45:5)
    at Object.<anonymous> (/Users/me/app/src/index.js:8:1)
```

겁먹지 마세요. 세 부분으로 분해됩니다:

### 부분 1 — 에러 종류와 메시지
```
TypeError: Cannot read properties of undefined (reading 'name')
```
- **TypeError**: 에러의 종류 (이 레슨 02번에서 종류별로 다룸)
- **Cannot read properties of undefined (reading 'name')**: 무슨 일이 일어났는지 — "undefined인 무언가에서 'name' 속성을 읽으려고 했다"

### 부분 2 — 스택 트레이스 (어디서 났는지)
```
    at getUser (/Users/me/app/src/users.js:12:18)
    at processData (/Users/me/app/src/main.js:45:5)
    at Object.<anonymous> (/Users/me/app/src/index.js:8:1)
```

위에서 아래로 읽으면 호출의 역순:
- 가장 위 `getUser`가 **에러가 직접 발생한 곳**
- 그 아래는 "그 함수를 누가 불렀나"의 추적

**가장 위 줄을 먼저 보세요.** `users.js:12:18` = users.js 파일 12번째 줄, 18번째 글자.

## 2) 가장 위 줄이 본인 코드가 아닐 때

```
    at axios.get (/node_modules/axios/lib/...)
    at getUser (/src/users.js:12:18)        ← 여기!
```

`node_modules` 안의 줄은 건너뛰세요. 보통 라이브러리 잘못이 아니라 **우리가 라이브러리에 잘못된 입력을 준 것**입니다. 본인 코드의 가장 위 줄에서 무엇을 호출했는지 보세요.

## 3) 브라우저 콘솔의 에러

브라우저 개발자 도구(F12)의 콘솔도 같은 형식:
```
Uncaught TypeError: Cannot read properties of null (reading 'click')
    at HTMLButtonElement.<anonymous> (app.js:23:14)
```

`app.js:23:14`을 클릭하면 그 줄로 바로 이동합니다.

## 4) Python의 에러

```
Traceback (most recent call last):
  File "main.py", line 8, in <module>
    process_data(users)
  File "main.py", line 4, in process_data
    return [u.name for u in users]
AttributeError: 'NoneType' object has no attribute 'name'
```

JavaScript와 반대로, **가장 아래 줄이 진짜 에러**입니다. 위로 올라갈수록 호출자.

## 5) 컴파일/빌드 에러는 다르게 생김

```
ERROR in ./src/App.jsx 15:7-12
Module not found: Error: Can't resolve './Button' in '/src'
```

이건 코드를 빌드하다가 난 에러. 보통:
- **Module not found**: import 경로가 틀림
- **Unexpected token**: 문법 오타 (대괄호 안 닫힘 등)
- **Cannot find name 'X'**: 정의 안 된 변수 사용 (TypeScript)

빌드 에러는 **실행 전**에 잡히고, 보통 메시지가 더 명확합니다.

---

## 에러를 읽는 3단계

1. **에러 종류** (TypeError? ReferenceError? SyntaxError?) — 다음 레슨에서 분류
2. **메시지** ("Cannot read properties of undefined") — 무엇이 일어났나
3. **첫 번째 본인 코드 줄 번호** — 어디서 났나

이 셋만 잡으면 80% 해결됩니다.

---

## AI에게 에러 던질 때 — 절대 하지 말 것

❌ "안 돼"
❌ "에러 났어"
❌ "고쳐줘"

이런 메시지로는 AI도 짐작만 합니다.

✅ **에러 메시지 전체 + 그 직전에 한 일** 둘 다 붙여서 던지세요.

```
[users.js:12 실행 중에] 다음 에러가 났어:

TypeError: Cannot read properties of undefined (reading 'name')
    at getUser (/src/users.js:12:18)
    at processData (/src/main.js:45:5)

users.js:12 줄은:
    return user.name

직전에 `getUser(undefined)`로 호출됐어. 어떻게 고쳐?
```

이러면 AI가 정확히 어디 어떻게를 짚을 수 있습니다. 다음 레슨에서 더 다룹니다.

ReferenceError: userId is not defined
    at validateInput (/app/src/auth.js:34:12)
    at handleLogin (/app/src/login.js:67:3)

(b) `auth.js`의 34번째 줄

</details>
