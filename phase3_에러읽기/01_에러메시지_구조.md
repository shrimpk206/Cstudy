# 01. 에러 메시지의 구조 🔍

## 한 줄로 말하면

에러는 무서운 게 아니에요.
**경찰서 보고서**같은 거예요. **단서가 다 있어요.** 읽는 법만 알면 됩니다.

---

## 🚨 사건 보고서 비유

범죄 현장에 가면 형사가:

1. **무슨 일이 일어났는지** 적어요 (TypeError, ReferenceError…)
2. **어디서 일어났는지** 적어요 (파일.js:12)
3. **누가 누굴 불렀는지** 적어요 (스택 트레이스)

에러 메시지가 정확히 이거예요!

---

## 1) 전형적인 에러 메시지 분해

```
TypeError: Cannot read properties of undefined (reading 'name')
    at getUser (/Users/me/app/src/users.js:12:18)
    at processData (/Users/me/app/src/main.js:45:5)
    at Object.<anonymous> (/Users/me/app/src/index.js:8:1)
```

겁먹지 마세요! **세 부분**으로 나눠보면:

### 🚨 부분 1 — 무슨 사건?

```
TypeError: Cannot read properties of undefined (reading 'name')
```

- **TypeError** = 사건 종류
- **Cannot read properties of undefined (reading 'name')** = "undefined인 뭔가에서 'name'을 꺼내려고 했음"

### 🗺️ 부분 2 — 스택 트레이스 (어디서?)

```
    at getUser (.../users.js:12:18)        ← 사건 발생 지점!
    at processData (.../main.js:45:5)       ← 그 함수를 부른 곳
    at Object.<anonymous> (.../index.js:8:1) ← 더 위
```

**JavaScript는 가장 위가 진짜 사건 현장!** 아래로 갈수록 "이 함수를 누가 불렀나"의 거꾸로 추적이에요.

`users.js:12:18` = users.js 파일, 12번째 줄, 18번째 글자.

---

## 2) 가장 위 줄이 본인 코드가 아닐 때

```
    at axios.get (/node_modules/axios/lib/...)   ← 이건 라이브러리
    at getUser (/src/users.js:12:18)              ← 여기! 본인 코드!
```

`node_modules` 안의 줄은 **건너뛰세요**. 보통 라이브러리 잘못이 아니라 **우리가 라이브러리에 잘못된 입력**을 준 것이에요.

> 🔍 **본인 코드의 첫 등장 줄이 디버깅의 시작점.**

---

## 3) 브라우저 콘솔 에러도 똑같아요

브라우저 개발자 도구(F12) → Console:

```
Uncaught TypeError: Cannot read properties of null (reading 'click')
    at HTMLButtonElement.<anonymous> (app.js:23:14)
```

`app.js:23:14`을 클릭하면 그 줄로 바로 이동! 진짜 편해요.

---

## 4) Python의 에러는 반대 방향! ⚠️

```
Traceback (most recent call last):
  File "main.py", line 8, in <module>
    process_data(users)
  File "main.py", line 4, in process_data
    return [u.name for u in users]
AttributeError: 'NoneType' object has no attribute 'name'
```

**Python은 가장 아래 줄이 진짜 에러!** JavaScript와 반대.

위로 올라갈수록 "어떤 호출 경로로 여기까지 왔나"의 역사예요.

---

## 5) 빌드 에러는 모양이 좀 달라요

```
ERROR in ./src/App.jsx 15:7-12
Module not found: Error: Can't resolve './Button' in '/src'
```

코드 실행 **전**에 잡히는 에러예요. 보통:

| 메시지 | 원인 |
|---|---|
| **Module not found** | import 경로 틀림 |
| **Unexpected token** | 문법 오타 (괄호 안 닫힘 등) |
| **Cannot find name 'X'** | 정의 안 된 변수 (TypeScript) |

빌드 에러는 메시지가 **더 명확**해서 사실 더 쉬워요!

---

## 🎯 에러 읽는 3단계 — 이거만 외우면 됩니다

1. **무슨 종류?** (TypeError? ReferenceError? SyntaxError?)
2. **메시지 뭐래?** ("Cannot read properties of undefined")
3. **어디 줄?** (본인 코드의 첫 등장 줄 번호)

이 셋만 잡으면 80% 해결!

🚫 "안 돼"

✅ **에러 메시지 전체 + 직전에 한 일** 함께 던지세요!

```
[users.js:12 실행 중에] 다음 에러:

TypeError: Cannot read properties of undefined (reading 'name')
    at getUser (/src/users.js:12:18)
    at processData (/src/main.js:45:5)

users.js:12 줄:
    return user.name

직전에 `getUser(undefined)`로 호출됨. 어떻게 고쳐?
```

이러면 AI가 **정확히** 짚어 고쳐줘요. 다음 레슨에서 더 자세히!
