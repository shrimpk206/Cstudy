# 01. 좋은 명세 쓰는 법 🍢

## 한 줄로 말하면

**떡볶이 주문**과 똑같아요.
"떡볶이 주세요"보다 "단맛, 곱빼기, 어묵 빼고, 매운 맛 5단계"가 백배 정확해요.

vibe coding 결과물 품질 = **첫 명세의 정확도.** 시작이 모호하면 끝까지 헤매요.

---

## 🍢 떡볶이 비유

분식집에서:

❌ "떡볶이 주세요"
→ 단맛인지 매운맛인지, 곱빼기인지 일반인지 알 수가 없죠.

✅ "단맛 떡볶이 곱빼기, 어묵 빼고 사리 추가"
→ 한 번에 정확히!

vibe coding도 똑같아요.

---

## 🚫 나쁜 명세 vs ✅ 좋은 명세

### ❌ 나쁨

> "할 일 관리 앱 만들어줘"

AI 머릿속:
- 무슨 기능들? ❓
- 어떻게 생김? ❓
- 데이터 어디에 저장? ❓
- 사용자 누구? ❓

→ AI가 추측으로 채움 → 본인 의도와 빗나감 😢

### ✅ 좋음

> 다음 사양의 할 일 관리 웹앱 만들어줘.
> 
> **사용자:** 개인이 혼자 쓰는 용도. 로그인 없음.
> 
> **기능:**
> 1. 할 일 추가 (텍스트만, 우선순위 같은 부가정보 X)
> 2. 할 일 완료 표시 (체크박스)
> 3. 완료된 항목 일괄 삭제 버튼
> 4. 새로고침해도 데이터 유지
> 
> **저장:** 브라우저 localStorage. 백엔드 없음.
> 
> **기술:** React + TypeScript + Tailwind CSS.
> 
> **UI:** 단일 페이지. 상단 입력창, 아래 목록. 다크 모드 X.
> 
> **성공 기준:**
> - 텍스트 입력 후 Enter로 추가
> - 체크박스로 완료 시 즉시 회색+취소선
> - 새로고침 후에도 데이터 그대로
> 
> 시작은 단일 컴포넌트로, 100줄 이내.

---

## 🧰 좋은 명세 7가지 재료 (떡볶이 8첩 반상!)

### 1️⃣ 사용자 — 누가 먹나
- 본인만? 여러 명?
- 로그인 필요?
- 모바일 / 데스크톱?

### 2️⃣ 핵심 기능 — 번호 매겨서 5개 이하
한 줄씩 짧게! 많으면 단계별로 나누기.

### 3️⃣ 빼는 것 — ★ 매우 중요!

> "검색 기능은 지금 빼"
> "다국어 지원 안 함"
> "관리자 페이지 없음"

AI는 시키지 않으면 자꾸 부풀려요. **하지 말 것**을 알려주는 게 더 효과적!

### 4️⃣ 데이터 — 어디에 저장
- localStorage (브라우저)
- 파일 (Node.js)
- 데이터베이스 (Postgres, MongoDB, SQLite, Firestore)
- 메모리만 (재시작하면 날아감)

### 5️⃣ 기술 스택
- 언어: JavaScript / TypeScript / Python
- 프레임워크: React / Next.js / Vue / Express
- 스타일: Tailwind / CSS

> 💡 **모르겠다면**: "추천해줘. AI가 잘 만드는 흔한 스택으로."

### 6️⃣ UI — 어떻게 생겼나
- 화면 개수
- 페이지 구조 (헤더, 사이드바, 본문...)
- **참고 이미지 / 사이트** 첨부 ("Notion처럼", "Linear처럼")
- 색감, 다크 모드 여부

### 7️⃣ 성공 기준 — 뭐가 되면 끝?
**체크 가능한** 형태로:

✅ "Enter 키로 추가됨"
✅ "새로고침해도 데이터 남음"
✅ "에러 없이 빌드됨"

❌ "예쁘게"
❌ "사용자가 만족"
❌ "잘 동작"

이게 없으면 **"다 됐어요"라는 AI 말을 검증할 방법이 없어요!**

---

## 💡 명세를 한 번에 다 적지 마세요

긴 명세를 한 번에 던지면 AI가 핵심 놓쳐요. 추천 흐름:

1. **첫 메시지**: 기능 1~2개 + 핵심 사용자 + 스택만
2. AI가 만든 결과 확인
3. 다음 기능 추가 요청
4. 반복

**작게 시작 → 동작 확인 → 추가.** vibe coding의 안정 패턴!

---

## 🤝 명세를 AI랑 같이 만들기

자신 없으면 명세 자체를 AI랑 같이 만들 수 있어요:

> 다음을 만들고 싶어: [한 문단 설명].
> 본격 작업 전에 이 명세에서 빠진 부분, 모호한 부분 질문해줘.
> 결정해야 할 것 5개 추려서 보여줘.

AI가 5개 질문 던지면 → 답변 → 그 답변까지 합쳐서 다시 명세 정리 요청!

## 🎯 확인해봐요

다음 명세에 **빠진 부분 3개** 이상 찾아보세요.

> "포트폴리오 사이트 만들어줘. 내 프로젝트들 보여주는 페이지 있고, 연락처도 있고. 멋있게."

너무 많이 빠짐:
- ❓ 기술 스택 (Next.js? Astro? 정적 HTML?)
- ❓ 프로젝트 데이터 입력/관리 (마크다운? JSON? CMS?)
- ❓ "멋있게"의 정의 (참고 사이트, 다크/라이트, 색감)
- ❓ 페이지 개수 (하나? 프로젝트별 상세?)
- ❓ 연락처 폼은 진짜 메일 발송? 아니면 mailto 링크?
- ❓ 모바일 우선 여부
- ❓ 배포 (Vercel? GitHub Pages?)
- ❓ 다국어 지원

좋은 시작점: 위 모든 걸 사전에 정해서 명시. 또는 AI에게 "이 명세에서 결정 필요한 부분 5개 질문해줘".
