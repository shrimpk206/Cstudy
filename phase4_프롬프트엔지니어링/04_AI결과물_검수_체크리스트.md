# 04. AI 결과물 검수 체크리스트

## 핵심 한 줄

"동작한다 = 올바르다"가 아닙니다. AI 코드는 **돌아가지만 위험한** 코드를 자주 만듭니다. 머지/배포 전에 이 체크리스트로 검토.

---

## 🔒 보안 (가장 중요)

- [ ] **API 키, 비밀번호, 토큰이 코드에 박혀있지 않은가?**
  - 키는 `.env`에, 코드는 `process.env.X`로
  - 프론트엔드 코드(React, Vue)에 비밀 키 들어가면 즉시 털림

- [ ] **`.gitignore`에 `.env`가 있는가?**

- [ ] **사용자 입력을 그대로 DB/HTML에 넣고 있지 않은가?**
  - SQL 인젝션: 파라미터화된 쿼리 (`?` 또는 `$1`) 써야 함
  - XSS: HTML에 사용자 입력 직접 삽입 금지 (React/Vue는 기본적으로 안전, 단 `dangerouslySetInnerHTML` 주의)

- [ ] **인증 / 권한 검사가 모든 보호된 엔드포인트에 있는가?**
  - 새 API 만들 때 자주 빼먹음
  - "/api/users/delete"가 누구나 호출 가능하면 끝

---

## 🐛 흔한 버그

- [ ] **`null` / `undefined` 처리가 됐나?**
  - 외부 데이터(API, 입력)는 항상 빈 값을 의심
  - `data?.field` 또는 명시적 검사

- [ ] **에러 처리가 있나?**
  - `await` 호출은 `try/catch`로 감싸기
  - 실패 시 사용자에게 뭐라고 보여줄지 정해져 있나

- [ ] **무한 루프 / 무한 재귀가 없나?**
  - `while`의 종료 조건
  - React의 `useEffect`에서 의존성 배열로 인한 무한 렌더링

- [ ] **숫자/문자 타입이 섞이지 않았나?**
  - 폼 입력은 항상 string. 계산 전 `Number()` 변환

- [ ] **비동기 함수에 `await`이 빠진 곳 없나?**

## 🎨 코드 품질

- [ ] **함수가 한 가지 일만 하는가?**
  - 200줄짜리 함수는 보통 너무 많은 일을 함

- [ ] **변수 이름이 의미가 있나?**
  - `data`, `temp`, `x` 같은 이름이 많으면 다시 짜달라고

- [ ] **같은 코드가 두세 군데 반복되어 있지 않나?**
  - 함수로 빼야 할 신호

- [ ] **사용하지 않는 import / 변수가 있나?**
  - 에디터의 ESLint가 잡아줌

- [ ] **AI가 자기가 안 쓰는 라이브러리를 추가하지 않았나?**

## ⚡ 성능 — 흔한 함정

- [ ] **반복문 안에서 API 호출 / DB 쿼리를 100번 하고 있지 않나?**
  - 한 번에 받아오는 방법이 거의 항상 있음

- [ ] **큰 데이터를 매번 처음부터 처리하고 있지 않나?**
  - 메모이제이션, 캐싱 고려

- [ ] **React에서 매 렌더마다 새 객체/함수를 만들어 자식에 넘기고 있지 않나?**
  - `useMemo`, `useCallback`이 필요한 시그널

## AI에게 검수 시키기

본인이 다 못 봐도 AI에게 시킬 수 있습니다:

> 위 코드에 대해 다음 관점으로 리뷰해줘:
> 1. 보안 취약점 (API 키 노출, XSS, 인증 누락)
> 2. 흔한 버그 (null 처리, 비동기 await 누락)
> 3. 성능 함정 (불필요한 반복 호출)
> 4. 사용되지 않는 코드
> 
> 각 항목마다 "발견 / 없음"으로 답하고, 발견이면 어디 어떻게 고칠지.

## 자가점검

다음 코드에서 문제 3가지를 찾아보세요.

```javascript
const apiKey = "sk-proj-Abc123..."

async function getData(userId) {
  const res = await fetch(`https://api.example.com/users/${userId}`, {
    headers: { Authorization: `Bearer ${apiKey}` }
  })
  return res.json()
}

document.getElementById("info").innerHTML = await getData(window.location.search)
```

<details>
<summary>정답</summary>

1. **API 키가 코드에 박혀있음** — 환경변수로 빼야. 게다가 이게 프론트엔드면 사용자에게 노출.
2. **`res.json()` 실패 처리 없음** — `try/catch` 또는 `if (!res.ok)` 검사 필요.
3. **XSS 위험** — `innerHTML`에 외부 데이터를 직접 넣음. 사용자가 `<script>` 태그 등을 삽입할 수 있음. `textContent` 또는 안전한 라이브러리 필요.
4. (보너스) `userId`로 `window.location.search`(쿼리 문자열 통째)를 그대로 넘김. URL 파싱 안 됐음.
</details>
