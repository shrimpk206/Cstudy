# 02. 컨텍스트 제공 — AI는 본인 머릿속을 못 봅니다

## 핵심 한 줄

AI는 본인의 프로젝트 구조, 사용자 의도, 기존 결정사항을 **모릅니다**. 매번 같이 던지세요. 같은 대화 안이라도 길어지면 잊습니다.

---

## 컨텍스트가 부족하면 일어나는 일

상황: 며칠 전 React 프로젝트를 만들었고, 오늘 새 기능을 추가하려고 합니다.

❌ 컨텍스트 없이:
> "댓글 기능 추가해줘"

→ AI가 무에서 만듦. 본인 프로젝트의 컴포넌트 구조, 스타일링 방식, 데이터 저장 방식 다 무시. 결국 통합 안 됨.

✅ 컨텍스트 같이:
> 현재 프로젝트는:
> - Next.js 14 App Router, TypeScript
> - 데이터는 Supabase
> - 스타일은 Tailwind + shadcn/ui
> - 인증은 next-auth (구글 로그인만)
> 
> 이미 있는 게시글 페이지(`/posts/[id]/page.tsx`)에 댓글 기능 추가해줘:
> - 로그인된 사용자만 작성 가능
> - 작성 시간 표시
> - 본인 댓글만 삭제 가능
> 
> 기존 게시글 페이지 코드:
> ```tsx
> [코드 붙여넣기]
> ```
> 
> Supabase에 `comments` 테이블이 이미 있어. 스키마:
> ```
> [스키마 붙여넣기]
> ```

## 컨텍스트 6종

### 1. 기술 스택
- 언어 / 프레임워크 / 버전
- 스타일링 방식
- 데이터베이스
- 인증 방식

### 2. 폴더/파일 구조
```
src/
├── components/
│   ├── ui/        ← shadcn 컴포넌트들
│   └── layout/
├── app/           ← Next.js App Router
└── lib/
    └── supabase.ts
```
긴 프로젝트면 일부만이라도. AI가 **어디에 새 파일을 둘지** 알 수 있게.

### 3. 컨벤션 / 스타일
- "함수는 화살표 함수로"
- "에러는 toast로 표시"
- "주석은 영어"
- "모든 컴포넌트는 named export"

### 4. 기존 결정사항
- "이전에 인증을 next-auth로 정했어"
- "에러 처리는 try/catch + sonner 토스트로 통일"

### 5. 관련 코드
새로 만들 게 아니라 **수정**할 거면 기존 파일 코드를 같이 던지세요.

### 6. 외부 자원
- API 문서 링크
- 디자인 시스템 (Figma URL)
- 이미 있는 데이터베이스 스키마

## 도구별 컨텍스트 자동화

### Cursor / Claude Code (CLI)
폴더 안에서 돌리면 **파일 자동 검색**이 됩니다. "이 프로젝트의 X 파일 찾아서 Y 추가해줘"식으로 가능.

### Cursor의 `@` 멘션
```
@components/Button.tsx 의 props에 size 추가해줘
```
파일을 명시적으로 참조.

### `.cursorrules` / `CLAUDE.md`
프로젝트 루트에 이런 파일을 두면 모든 대화에 자동 포함:

```markdown
# 프로젝트 규칙

- TypeScript strict 모드
- 함수형 컴포넌트, named export
- 에러는 sonner 토스트로
- 데이터베이스는 Supabase, RLS 활성화
- 새 라이브러리 추가 전 기존 의존성으로 가능한지 확인
```

→ 매번 컨벤션 다시 안 적어도 됨. **vibe coding 효율의 핵심**.

---

## 대화가 길어지면 컨텍스트가 흐려진다

같은 대화 안이라도 50번 주고받으면 AI가 초반에 정한 것을 잊을 수 있습니다.

**대처:**
- 새 큰 기능 시작할 때마다 핵심 컨텍스트 다시 요약해서 던지기
- 또는 새 대화 시작 (Cursor의 새 채팅, Claude Code의 `/clear`)
- "지금까지의 결정사항을 요약해서 메모로 만들어줘" → 그 메모를 다음 대화 첫 메시지로
