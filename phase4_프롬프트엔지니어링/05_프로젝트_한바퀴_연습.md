# 05. 프로젝트 한 바퀴 — 모든 걸 종합한 실습

## 졸업 시험

이 프로그램에서 배운 모든 걸 한 프로젝트에 적용해보는 연습입니다. 실제로 vibe coding으로 작은 앱 하나를 끝까지 만들어보세요.

---

## 추천 프로젝트 — 4가지 중 택 1

본인 수준과 흥미에 맞춰 고르세요.

### A. 메모 앱 (난이도 ★)
- 텍스트 입력 → 목록에 추가 → localStorage 저장
- 삭제, 검색
- 단일 페이지

### B. 환율 변환기 (난이도 ★★)
- 외부 API에서 환율 가져오기 (예: exchangerate-api)
- 원화 → 다른 화폐 변환
- 자주 쓰는 화폐 즐겨찾기 (localStorage)
- API 키를 환경변수로 분리하는 연습

### C. 할 일 관리 with Supabase (난이도 ★★★)
- 구글 로그인
- 할 일 추가/완료/삭제
- 다른 기기에서도 동기화
- 환경변수, 인증, 데이터베이스 모두 다룸

### D. 본인이 vibe coding으로 늘 만들고 싶었던 것 (난이도 ?)
- 자유. 단, **하루 안에 만들 수 있는 크기**로 잘게 자르세요

---

## 진행 단계 — 모든 학습 적용

### 단계 0: 명세 작성 (`phase4/01`)
프로젝트 시작 전 다음을 적어두세요. 가급적 종이/메모장에:

```
프로젝트: ___
사용자: ___
핵심 기능 (5개 이하):
1.
2.
...
빼는 것:
- 
기술 스택: ___
저장: ___
성공 기준:
1.
2.
3.
```

이걸 첫 프롬프트로 그대로 던집니다.

### 단계 1: 첫 코드 받기 (`phase4/02`)
컨텍스트와 함께 명세를 던지고 가장 단순한 첫 버전을 만들게 합니다:

> 위 명세대로 만들 거야. 첫 단계로 [핵심 기능 1개]만 동작하는 가장 단순한 버전을 만들어줘. 50줄 이내로.

→ 받은 코드를 직접 실행해보세요.

### 단계 2: 코드 읽기 (`phase2/*`)
실행 전 코드를 한 번 훑으세요. **다음을 짚어보세요:**
- 변수, 함수, 조건문, 반복문이 각각 어디인가
- import는 뭐가 됐고 어디서 쓰이는가
- 외부 호출(API, DB)은 어디인가
- 비동기는 어디인가

모르는 단어가 있으면 `phase2/어휘_치트시트.md` 검색.

### 단계 3: 첫 에러 처리 (`phase3/*`)
거의 확실히 에러가 납니다. 각 에러마다:
1. 에러 종류와 줄 번호 짚기 (`phase3/01`)
2. 어떤 종류 에러인지 분류 (`phase3/02`)
3. 좋은 형식으로 AI에게 다시 던지기 (`phase3/03`)

**에러 일지를 적어보세요:**
- 만난 에러 종류 (TypeError 5번, ModuleNotFound 2번 등)
- 각 에러를 어떻게 풀었는지

### 단계 4: 기능 추가 (`phase4/02-03`)
첫 기능이 동작하면 다음 기능 추가. 한 번에 하나씩.
- 잘 안 되면 `phase4/03`의 재시도 패턴 활용
- 컨텍스트 유지하기

### 단계 5: 검수 (`phase4/04`)
모든 기능 완성 후 `phase4/04`의 체크리스트로 점검:
- 보안: API 키, .env, .gitignore
- 흔한 버그: null 처리, 에러 처리
- 코드 품질: 함수 분리, 중복 제거

AI에게도 검수 요청해서 본인이 놓친 것 잡기.

### 단계 6: 회고
프로젝트 끝나고 다음을 메모:
- 가장 시간 많이 든 부분
- 가장 자주 만난 에러 종류
- 다음 프로젝트에서 시작 단계에 더 명확히 할 것
- 학습 자료에서 부족했던 부분

---

## 첫 프로젝트 회고를 본인 학습 자료에 추가

`codingstudy/회고/01_첫프로젝트.md` 같은 파일을 직접 만들어서:

```markdown
# 첫 프로젝트 회고 — [프로젝트명]

## 만든 것
- 
## 만나서 배운 에러
- TypeError ×3, 원인은 ...
- ...

## 다음에 더 잘 할 것
- 명세 단계에서 X를 빠뜨리지 말 것
- ...

## 새로 알게 된 어휘
- 
```

이게 누적되면 본인만의 vibe coding 매뉴얼이 됩니다.
