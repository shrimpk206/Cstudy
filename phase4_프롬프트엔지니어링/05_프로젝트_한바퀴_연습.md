# 05. 프로젝트 한 바퀴 — 졸업 모험! 🏆

## 🏰 졸업 시험!

이 프로그램에서 배운 모든 걸 한 프로젝트에 적용해보는 모험이에요. 작은 앱 하나를 vibe coding으로 끝까지 만들어보세요.

---

## 🎯 추천 프로젝트 — 4가지 중 택 1

본인 수준과 흥미로 골라요.

### 🌱 A. 메모 앱 (난이도 ★)

- 텍스트 입력 → 목록 추가 → localStorage 저장
- 삭제, 검색
- 단일 페이지

### 💱 B. 환율 변환기 (난이도 ★★)

- 외부 API에서 환율 가져오기 (예: exchangerate-api)
- 원화 → 다른 화폐
- 자주 쓰는 화폐 즐겨찾기 (localStorage)
- API 키를 환경변수로 분리 연습

### ✅ C. 할 일 관리 with Supabase (난이도 ★★★)

- 구글 로그인
- 할 일 추가/완료/삭제
- 다른 기기에서도 동기화
- 환경변수, 인증, 데이터베이스 모두

### 🎮 D. 본인이 만들고 싶었던 것

- 자유! 단, **하루 안에 만들 수 있는 크기**로 잘게

---

## 🗺️ 모험 단계 — 모든 학습 적용!

### 🎬 단계 0: 명세 작성 (Phase 4-01)

종이/메모장에:

```
프로젝트: ___
사용자: ___
핵심 기능 (5개 이하):
1.
2.
...
빼는 것:
- 
기술 스택: ___
저장: ___
성공 기준:
1.
2.
3.
```

이걸 첫 프롬프트로 그대로!

---

### 🚀 단계 1: 첫 코드 받기 (Phase 4-02)

컨텍스트 + 명세 던지고 가장 단순한 첫 버전:

> 위 명세대로 만들 거야. 첫 단계로 [핵심 기능 1개]만 동작하는 가장 단순 버전. 50줄 이내로.

→ 받은 코드 직접 실행!

---

### 📖 단계 2: 코드 읽기 (Phase 2 어휘)

실행 전 코드 한 번 훑어요. **다음 짚기**:

- 변수, 함수, 조건문, 반복문 어디?
- import는 뭐가 됐고 어디 쓰이는지?
- 외부 호출(API, DB) 어디?
- 비동기 어디?

모르는 단어 → `phase2/어휘_치트시트.md` 검색!

---

### 🔧 단계 3: 첫 에러 처리 (Phase 3)

거의 확실히 에러가 나요. 각 에러:

1. 종류와 줄 번호 짚기 (Phase 3-01)
2. 어떤 종류 분류 (Phase 3-02)
3. 좋은 형식으로 AI에 던지기 (Phase 3-03)

**📔 에러 일지 적기:**
- 만난 에러 종류 (TypeError 5번, ModuleNotFound 2번 등)
- 각 에러를 어떻게 풀었나

---

### 🎁 단계 4: 기능 추가 (Phase 4-02, 03)

첫 기능 동작하면 다음 기능. **한 번에 하나씩.**

- 잘 안 되면 Phase 4-03의 재시도 패턴
- 컨텍스트 유지

---

### 🔍 단계 5: 검수 (Phase 4-04)

모든 기능 완성 후 체크리스트로 점검:

- 🔒 보안: API 키, .env, .gitignore
- 🐛 흔한 버그: null 처리, 에러 처리
- 🎨 코드 품질: 함수 분리, 중복 제거

AI에게도 검수 요청해서 본인이 놓친 것 잡기!

---

### 📓 단계 6: 회고 — 보물 일기 ⭐

프로젝트 끝나고 메모:
- 가장 시간 많이 든 부분
- 가장 자주 만난 에러 종류
- 다음 프로젝트에서 시작 단계에 더 명확히 할 것
- 학습 자료에서 부족했던 부분

---
