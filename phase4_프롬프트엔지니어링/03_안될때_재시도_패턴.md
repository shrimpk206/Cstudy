# 03. 안 될 때 재시도 패턴 🎮

## 한 줄로 말하면

**게임 보스전**과 비슷해요.
같은 공격만 반복하면 절대 못 깨요. **다른 길**로 가야 해요.

"다시 해줘"는 거의 같은 결과를 만들어요. 다른 결과를 원하면 **다른 정보**를 던지거나 **다른 각도**에서 물어요.

---

## 🎮 보스전 비유

게임에서 보스 못 깨겠으면:

1. **무기 바꿔보기** (다른 각도)
2. **친구한테 공략 물어보기** (외부 정보)
3. **레벨 더 키우기** (단순화)
4. **다시 시작** (롤백)

vibe coding 막혔을 때 9가지 패턴이 있어요!

---

## 패턴 1: "왜 그렇게 했는지 설명해줘" 🤔

코드 안 돌아갈 때 **즉시 다시 짜라고 시키지 말고**, 설명을 먼저 받으세요:

> 위 코드에서 X 부분을 왜 그렇게 했는지 한 줄씩 설명해줘. 그 다음 어디가 문제일지 가설 3개 줘.

→ 가끔 AI가 **자기 코드의 모순을 스스로 발견**해요. 본인도 코드를 더 잘 이해하게 돼요!

---

## 패턴 2: "무슨 가정이 틀렸는지" 🎯

같은 에러가 반복되면:

> 같은 에러가 또 나. 너가 가진 가정 중 어떤 게 틀렸을 가능성? 가설 3개, 각각 어떻게 검증할지.

→ AI가 **첫 가설에 갇히지 않고** 다른 방향 탐색.

---

## 패턴 3: "최소 재현" 🧪

복잡한 코드에서 못 찾으면 단순화:

> 이 컴포넌트에서 문제 나는데 어디 때문인지 모르겠어. 다른 모든 부분 빼고, 문제를 재현하는 가장 작은 코드 만들어줘. 거기서부터 디버깅하자.

→ 단순화 과정에서 **원인이 드러나요!**

---

## 패턴 4: "롤백하고 다른 방향" ⏪

AI가 미궁에 빠지면 차라리 되돌리는 게 빨라요:

> 마지막 두 번의 수정 되돌리고, 처음 동작했던 상태로. 다음 다른 접근으로 시작.

> 💡 **git 자주 commit해두세요!** 그래야 안전하게 되돌릴 수 있어요.

---

## 패턴 5: "본인이 짠 게 아닌 척" 👔

> 누군가 다른 사람이 짠 다음 코드를 받았다고 가정해. 잘못된 부분 5개 찾아줘:
> ```
> [방금 AI가 짠 코드]
> ```

→ 자기 코드 비판은 어색한지 AI가 **객관적인 눈**을 얻어요!

---

## 패턴 6: "더 단순하게" 🔧

AI가 자꾸 부풀릴 때:

> 이거 너무 복잡해. 추상화 다 빼고 50줄 이내의 가장 단순한 버전으로 다시. 나중에 필요하면 그때 추상화 추가.

→ **단순화는 강력한 패턴!** AI는 "제대로" 하려고 추상화 더하는 경향이 있어요.

---

## 패턴 7: "다른 라이브러리/접근" 🔄

> 지금 X 라이브러리로 시도 중인데 잘 안 돼. 같은 일을 Y 라이브러리로 하면? 어느 쪽이 더 단순한지 비교.

→ AI는 종종 **한 라이브러리에 갇혀요!**

---

## 패턴 8: "단계별로 나누기" 🪜

큰 변경이 매번 깨지면:

> 한 번에 다 하지 말고 작은 단계로:
> 1. X만 추가하고 동작 확인
> 2. 그 다음 Y 추가
> 3. 마지막으로 Z
> 
> 각 단계마다 멈추고 내가 확인할게.

---

## 패턴 9: "외부 정보로 재무장" 📚

> [Stack Overflow / GitHub 이슈 / 공식 문서]에서 이런 답 찾았어:
> 
> [내용 붙여넣기]
> 
> 우리 코드에 어떻게 적용?

→ 정답이 어딘가 있으면 AI가 **그걸 적용하는 데에 강해요!**

---

## 🚫 절대 하지 말 것

❌ 같은 프롬프트 5번 반복 → 같은 답
❌ 점점 짧아지는 짜증 메시지 ("아니 그게 아니라", "됐고") → AI가 정보 줄어 더 추측
❌ 에러 메시지를 본인 말로 요약 → 원본 정보 사라짐

---

## 🌊 막혔을 때 흐름

1. 같은 메시지 다시 던지기 ❌
2. **잠깐 멈추고** — 정확히 뭐가 안 되는지 본인이 한 줄로 적기
3. 위 패턴 중 하나 선택
4. 그래도 안 되면 → 검색 (Phase 3 - 04)
5. 검색 결과를 AI에게 다시 던지기

---
