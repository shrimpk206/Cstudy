# 05. 파일과 모듈 📦

## 한 줄로 말하면

코드를 한 파일에 다 쓰면 **지옥**이에요.
여러 파일로 쪼개고 **빌려와서** 써요.

---

## 📦 비유: 친구한테 빌려주고 받기

학교에서:
- 내가 가위를 가지고 있어요 → 친구한테 **빌려줘요** (export)
- 친구가 자를 가지고 있어요 → 나도 **빌려와요** (import)

코드도 똑같아요.

---

## 1) export — "빌려줄게!" 📤

`utils.js`:
```javascript
export function add(a, b) {
  return a + b
}

export const PI = 3.14
```

**`export`** = "다른 파일이 이거 가져다 써도 돼."

## 2) import — "빌려갈게!" 📥

`main.js`:
```javascript
import { add, PI } from "./utils.js"

console.log(add(3, 5))   // 8
console.log(PI)           // 3.14
```

해석:
- `{ add, PI }` ← 가져올 이름들 (중괄호 안에)
- `from "./utils.js"` ← 어디서? (이 폴더의 utils.js)

---

## 3) default export — "대표 한 명만 빌려줄게" 🌟

```javascript
// Button.js
export default function Button() {
  return <button>클릭</button>
}
```

```javascript
// App.js
import Button from "./Button.js"
//      ↑ 중괄호 없음! 이름도 자유!
//        import MyButton from "..." 도 OK
```

### 차이 정리

| | named export | default export |
|---|---|---|
| 만들 때 | `export function add() {}` | `export default function() {}` |
| 가져올 때 | `import { add } from ...` | `import 무엇이든 from ...` |
| 이름 | 그대로 써야 | 자유롭게 변경 가능 |

---

## 4) 외부 라이브러리도 같은 방식

```javascript
import axios from "axios"          // 외부 라이브러리 (./없음)
import { useState } from "react"   // react에서 useState만
```

> 💡 `./` 또는 `../`가 있으면 **내 파일**, 없으면 **외부 라이브러리**.

---

## 5) 경로 표기 — 길 찾기 🗺️

```
my-project/
├── components/
│   ├── Button.js
│   └── Card.js
└── utils/
    └── helper.js
```

`Button.js`에서:
- `"./Card"` → 같은 폴더의 Card
- `"../utils/helper"` → 상위 폴더의 utils 폴더의 helper
- `"react"` → 외부 라이브러리

> 🧭 `./` = 같은 층, `../` = 한 층 위로

---

## 6) Python의 import

거의 같아요:

```python
# utils.py
def add(a, b):
    return a + b

# main.py
from utils import add
import math

print(add(3, 5))
print(math.pi)
```

---

## 7) 두 가지 옛/새 방식 — 헷갈리면 안 돼요

### 옛날 (CommonJS) — Node.js 옛 코드

```javascript
const axios = require("axios")
module.exports = { add }
```

### 새 방식 (ES Modules) — 요즘 표준

```javascript
import axios from "axios"
export { add }
```

> ⚠️ 둘을 **섞으면 에러**! AI가 섞으면 "통일해줘"라고 하세요.

| 한 파일이 500줄 넘게 큼 | "이 파일을 기능별로 쪼개고 import로 연결해줘" |

| `require`와 `import`가 섞여있음 | "프로젝트를 ES Modules로 통일해줘" |

### 💡 검수 꿀팁

파일 맨 위 **import 줄들만 봐도** 이 파일이 뭘 쓰는지 한눈에 보여요. 의도와 맞는지 확인!

AI가 갑자기 처음 보는 라이브러리를 import하면 → "이거 진짜 필요해? 표준 기능으로 안 돼?" 한번 물어봐요.

## 🎯 확인해봐요

```javascript
import { foo, bar } from "./helper.js"
import React from "react"
```

1. `helper.js`에 어떤 export가 있어야 할까요?
2. `import React from "react"` — React는 default export일까요, named일까요?

<details>
<summary>👀 정답 보기</summary>

1. `export function foo(...)`와 `export ... bar` — **이름 그대로** export.
2. **default export**! 중괄호 `{}`가 없으니까.
