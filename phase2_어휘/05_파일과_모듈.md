# 05. 파일과 모듈 — import, export

## 핵심 한 줄

코드를 한 파일에 다 쓰면 지옥입니다. 여러 파일로 쪼개고, 필요할 때 **import**로 가져옵니다.

---

## 1) export — "이 함수를 다른 파일이 쓸 수 있게 내보낸다"

`utils.js`:
```javascript
export function add(a, b) {
  return a + b
}

export const PI = 3.14
```

## 2) import — "다른 파일에서 가져온다"

`main.js`:
```javascript
import { add, PI } from "./utils.js"

console.log(add(3, 5))   // 8
console.log(PI)           // 3.14
```

- `{ add, PI }` — 가져올 이름들 (중괄호 안에)
- `from "./utils.js"` — 어디서? (상대 경로)

## 3) default export — 파일에서 "대표 1개"만 내보내는 방식

`Button.js`:
```javascript
export default function Button() {
  return <button>클릭</button>
}
```

`App.js`:
```javascript
import Button from "./Button.js"   // 중괄호 없음! 이름도 자유롭게 바꿔도 됨
```

**name vs default 차이:**
- named export: `import { add } from ...` — 이름 그대로 써야 함
- default export: `import 무엇이든 from ...` — 이름 자유

## 4) 외부 라이브러리 import

```javascript
import axios from "axios"          // node_modules에 설치된 라이브러리
import { useState } from "react"   // React에서 useState만 가져오기
```

상대 경로(`./`)가 없으면 → "node_modules에 깔린 라이브러리"라는 뜻.

## 5) Python의 import

```python
# utils.py
def add(a, b):
    return a + b

# main.py
from utils import add
import math

print(add(3, 5))
print(math.pi)
```

개념은 같습니다, 문법만 다름.

## 6) 경로 표기

- `./utils` — 현재 폴더의 utils
- `../utils` — 상위 폴더의 utils
- `./components/Button` — 현재 폴더의 components 폴더 안의 Button
- `react` (./나 ../ 없음) — 외부 라이브러리

## 7) CommonJS vs ES Modules — 두 가지 문법

오래된 Node.js 코드는 이렇게 생겼습니다:
```javascript
// 옛날 방식 (CommonJS)
const axios = require("axios")
module.exports = { add }
```

새 방식 (ES Modules):
```javascript
import axios from "axios"
export { add }
```

AI가 둘을 섞으면 에러 납니다. 통일하라고 하세요.

## AI에게 적용

| 상황 | 이렇게 말하세요 |
|---|---|
| 한 파일이 500줄 넘게 길어짐 | "이 파일을 기능별로 여러 파일로 쪼개고 import로 연결해줘" |
| 같은 함수를 여러 파일에서 복사 붙여넣기 함 | "이 함수를 `utils.js`로 빼고 import해서 쓰게 해줘" |
| `Cannot find module 'xyz'` 에러 | "이 모듈이 설치 안 됐거나, 경로가 틀렸어. 어떤 거야?" |
| `require`와 `import`가 섞여 있음 | "프로젝트를 ES Modules로 통일해줘" (또는 그 반대) |

**검수 포인트:**
- 파일 맨 위 `import`들을 보면 이 파일이 뭘 쓰는지 한눈에 보입니다. 그게 의도와 맞는지 확인.
- AI가 갑자기 본 적 없는 라이브러리를 import하면 → "이 라이브러리 진짜 필요해? 표준 기능으로 안 돼?"

## 자가점검

1. `import { foo, bar } from "./helper.js"`에서 `helper.js`에 어떤 export가 있어야 할까?
2. `import React from "react"` — 여기서 React는 default export일까 named export일까?

<details>
<summary>정답</summary>

1. `export function foo(...)`와 `export ... bar` — 이름을 그대로 export해야 함.
2. **default export**. 중괄호가 없기 때문.
</details>
