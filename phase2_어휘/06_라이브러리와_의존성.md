# 06. 라이브러리와 의존성 🧱

## 한 줄로 말하면

남이 미리 만들어둔 **레고 키트**예요.
가져다 조립하면 빠르고 편해요.

---

## 🧱 레고 비유

집을 만들고 싶어요:

- 방법 A: 나무 베고, 못 깎고, 다 처음부터… 😩 1년 걸림
- 방법 B: **레고 키트** 사서 조립 → 1시간 ✨

vibe coding 결과물은 거의 다 **레고 조립**이에요.

- **라이브러리** = 레고 키트
- **의존성** = 우리 프로젝트가 그 키트 부품을 빌려 쓰는 것

---

## 1) 왜 라이브러리를 쓰나요?

날짜 계산을 직접 짜려면 끔찍해요. 다행히 누군가 `dayjs` 키트를 만들어뒀어요:

```javascript
import dayjs from "dayjs"
dayjs().format("YYYY-MM-DD")   // "2026-04-27"
```

3줄로 끝! **표준 라이브러리에 의존**하는 게 똑똑한 vibe coding이에요.

---

## 2) JavaScript: npm 🍱

```bash
npm install dayjs           # 라이브러리 설치 (도시락 받기)
npm install                 # package.json에 적힌 거 다 받기
npm uninstall dayjs         # 제거
```

### `package.json` = 키트 목록 📋

이 프로젝트가 어떤 키트를 쓰는지 적힌 명세서예요:

```json
{
  "name": "my-app",
  "dependencies": {
    "dayjs": "^1.11.10",
    "axios": "^1.6.0"
  },
  "devDependencies": {
    "vitest": "^1.0.0"
  }
}
```

### dependencies vs devDependencies

| | 뜻 | 예 |
|---|---|---|
| **dependencies** | 실행에 필요 | 라이브러리들 (axios, react) |
| **devDependencies** | 개발할 때만 | 테스트 도구, 타입 검사 |

배포할 때 `dev`는 안 들어가요. **개발 도구는 dev**, **앱이 쓰는 건 dependencies**.

### `node_modules/` 폴더

실제 다운로드된 키트 부품들이 여기 있어요. **엄청 큼!** 깃에는 안 올려요. 다른 사람은 `npm install` 한 번이면 다 받을 수 있으니까.

### `package-lock.json`

정확히 어떤 버전이 깔렸는지 잠금. **이건 깃에 올려요.**

## 3) Python: pip 🐍

```bash
pip install requests
pip install -r requirements.txt   # 적힌 거 다 받기
```

`requirements.txt`:
```
requests==2.31.0
pandas>=2.0.0
```

또는 더 새 방식 `pyproject.toml`도 있어요.

---

## 4) 가상환경 — Python에서 특히 중요 🏠

프로젝트마다 키트를 격리시키는 거예요. 한 컴퓨터에서 여러 프로젝트가 안 섞이게.

```bash
python -m venv venv          # venv 폴더에 가상환경 만들기
source venv/bin/activate     # Mac/Linux
venv\Scripts\activate        # Windows
pip install ...              # 이 가상환경에만 설치됨
```

JavaScript는 `node_modules`가 폴더별로 자동 분리되어 가상환경 개념이 덜 필요해요.

## 5) 버전 표기법 — `^` vs `~` vs 정확히

`1.11.10`은 **메이저.마이너.패치**예요.

| 변경 | 의미 | 예 |
|---|---|---|
| **메이저** | 큰 변화, 호환성 깨짐 | 1 → 2 |
| **마이너** | 새 기능 | 11 → 12 |
| **패치** | 버그 수정 | 10 → 11 |

| 표기 | 어디까지 자동 업데이트? |
|---|---|
| `^1.11.10` | 마이너/패치 (1.x.x 안에서) |
| `~1.11.10` | 패치만 (1.11.x) |
| `1.11.10` | 그것만! |

> 💡 보통 `^`가 기본. 큰 사고 안 나게 메이저는 고정.

|---|---|
| `Cannot find module 'xyz'` 에러 | "이거 설치 안 됐어. 명령어 알려줘" → AI가 `npm install xyz` |
