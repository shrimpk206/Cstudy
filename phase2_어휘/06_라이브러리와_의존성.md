# 06. 라이브러리와 의존성 — npm, pip, package.json

## 핵심 한 줄

남이 미리 만들어둔 코드 묶음 = **라이브러리**. 이걸 가져다 쓰는 게 **의존성(dependency)**. 우리 프로젝트가 그 라이브러리에 "의존"한다는 뜻.

---

## 1) 왜 라이브러리를 쓰나

날짜 계산을 직접 짜려면 끔찍합니다. 다행히 누군가 `dayjs`라는 라이브러리를 만들어뒀습니다.

```javascript
import dayjs from "dayjs"
dayjs().format("YYYY-MM-DD")   // "2026-04-27"
```

vibe coding의 결과물은 거의 다 라이브러리 조립입니다.

## 2) JavaScript: npm

```bash
npm install dayjs           # 라이브러리 설치
npm install                 # package.json에 적힌 거 다 설치
npm uninstall dayjs         # 제거
```

**`package.json`** = 이 프로젝트가 뭘 의존하는지 적힌 명세서:
```json
{
  "name": "my-app",
  "dependencies": {
    "dayjs": "^1.11.10",
    "axios": "^1.6.0"
  },
  "devDependencies": {
    "vitest": "^1.0.0"
  }
}
```

- `dependencies`: 실행에 필요한 것
- `devDependencies`: 개발할 때만 필요한 것 (테스트, 타입 검사 등)
- `^1.11.10`: 1.11.10 이상, 2.0.0 미만 (마이너 업데이트는 자동)

**`node_modules/` 폴더** = 실제로 다운로드된 라이브러리 코드들. 매우 거대해서 git에는 안 올리고, 다른 사람이 받으면 `npm install` 한 번으로 복원합니다.

**`package-lock.json`** = 정확히 어떤 버전이 깔렸는지 기록. 이건 git에 올립니다.

## 3) Python: pip

```bash
pip install requests
pip install -r requirements.txt   # requirements.txt에 적힌 거 다 설치
```

`requirements.txt`:
```
requests==2.31.0
pandas>=2.0.0
```

또는 `pyproject.toml`을 쓰는 더 새 방식도 있습니다.

## 4) 가상환경 (virtual environment)

Python에서 특히 중요. 프로젝트마다 라이브러리를 격리시키는 것:
```bash
python -m venv venv          # venv 폴더에 가상환경 생성
source venv/bin/activate     # Mac/Linux
venv\Scripts\activate        # Windows
pip install ...              # 이 가상환경에만 설치됨
```

JavaScript는 `node_modules`가 폴더별로 자동 분리되어 가상환경 개념이 덜 필요합니다.

## 5) 버전 표기법 (semver)

`1.11.10` = `메이저.마이너.패치`
- **메이저** 1 → 2: 호환성 깨짐, 코드 수정 필요할 수 있음
- **마이너** 11 → 12: 새 기능 추가, 기존 코드 그대로 동작
- **패치** 10 → 11: 버그 수정만

`^1.11.10`은 마이너/패치 자동 업데이트. `~1.11.10`은 패치만. `1.11.10`은 그것만.

## AI에게 적용

| 상황 | 이렇게 말하세요 |
|---|---|
| `Cannot find module 'xyz'` 에러 | "이거 설치 안 됐어. 설치 명령어 알려줘" → AI가 `npm install xyz` 알려줌 |
| 똑같이 만든다고 하는데 라이브러리를 자꾸 다른 걸 씀 | "이미 `dayjs` 깔려있으니 새 라이브러리 추가하지 말고 이걸로 해줘" |
| package.json이 너무 커지고 안 쓰는 라이브러리가 많음 | "package.json에서 코드에 import 안 된 라이브러리 찾아줘" |
| 라이브러리 업데이트 후 깨짐 | "이거 어느 버전부터 안 되지? 메이저 업데이트 됐는지 확인해줘" |

**검수 포인트:**
- AI가 매우 마이너한 라이브러리(다운로드 수 적은)를 추천하면 의심하세요. 보안 위험이 있을 수 있습니다.
- "그 기능, 표준 자바스크립트로 안 돼?"라고 한번 물어보면 라이브러리 안 깔아도 되는 경우가 많습니다.
