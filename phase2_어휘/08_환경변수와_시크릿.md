# 08. 환경변수와 시크릿 — .env, API 키

## 핵심 한 줄

API 키나 비밀번호를 코드에 직접 적으면 깃허브에 올라가는 순간 털립니다. **환경변수**에 분리해서 관리합니다.

---

## 1) 무엇이 시크릿인가

- API 키 (OpenAI, Stripe, AWS 등)
- 데이터베이스 비밀번호
- 인증 토큰
- 결제 시크릿

이건 **절대로** 코드 파일에 직접 적지 마세요.

```javascript
// ❌ 절대 이렇게 하지 말 것
const apiKey = "sk-proj-abc123..."

// ✅ 환경변수에서 읽기
const apiKey = process.env.OPENAI_API_KEY
```

## 2) `.env` 파일

프로젝트 루트에 `.env` 파일을 만듭니다:
```
OPENAI_API_KEY=sk-proj-abc123...
DATABASE_URL=postgres://user:pass@localhost/db
PORT=3000
```

**규칙:**
- `KEY=value` 형식
- 따옴표 없음 (또는 있어도 됨, 도구에 따라 다름)
- 한 줄에 하나
- `=` 양옆에 공백 없음

## 3) `.gitignore`에 `.env` 추가 — 가장 중요

```
# .gitignore
node_modules/
.env
.env.local
```

`.gitignore`는 git에 올리지 않을 파일 목록입니다. `.env`는 **반드시** 여기 들어가야 합니다.

## 4) 코드에서 사용

JavaScript:
```javascript
// dotenv 라이브러리 필요 (Node.js)
import "dotenv/config"

console.log(process.env.OPENAI_API_KEY)
```

Vite/Next.js 같은 프레임워크는 자동으로 `.env`를 로드합니다 (단, 프론트엔드 노출용은 `VITE_` 또는 `NEXT_PUBLIC_` 접두사 필요).

Python:
```python
import os
from dotenv import load_dotenv

load_dotenv()
api_key = os.getenv("OPENAI_API_KEY")
```

## 5) `.env.example`

다른 사람이 프로젝트를 받을 때 어떤 환경변수가 필요한지 알려주는 파일:
```
# .env.example  (이건 git에 올림)
OPENAI_API_KEY=
DATABASE_URL=
PORT=3000
```
값은 비어두고, 키 이름만 적어둡니다.

## 6) 시크릿이 이미 깃허브에 올라갔다면

1. **즉시 그 키를 폐기(rotate)**하세요. 서비스 대시보드에서 키를 새로 발급받고 옛 키를 무효화.
2. git history에서 지운다고 해도 누가 이미 봤을 수 있습니다. 폐기가 1순위.
3. 그 다음 `.gitignore`에 추가하고 다시는 올라가지 않게 합니다.

## AI에게 적용

| 상황 | 이렇게 말하세요 |
|---|---|
| AI가 코드에 API 키를 박아넣었음 | "이 키를 환경변수로 빼줘. `.env` 파일과 `.gitignore`도 만들어줘" |
| 새 API를 연동함 | "이 키는 `.env`에 넣고 `.env.example`에도 빈 값으로 추가해줘" |
| 클라이언트(브라우저)에서 키를 쓰려고 함 | "잠깐, 이 키 브라우저에 노출되면 위험한 거 아냐? 서버에서만 쓰게 해줘" ← 매우 중요 |

**검수 포인트 — 가장 자주 실수하는 부분:**
- AI는 종종 React/Vue 같은 프론트엔드에서 API 키를 그대로 쓰려고 합니다. 프론트엔드에 들어가는 모든 코드는 사용자에게 보입니다. 비밀이 아닙니다.
- 진짜 비밀 키가 필요하면 **백엔드(서버)** 거쳐서 호출해야 합니다. AI에게 "프록시 엔드포인트 만들어줘"라고 하세요.

2. React 빌드 결과물은 사용자 브라우저로 다 전송됩니다. 키도 같이 노출됩니다. 백엔드 서버를 두고 거기서 OpenAI를 호출해야 합니다.
