# 01. 데이터를 담는 그릇 — 변수, 배열, 객체

## 핵심 한 줄

프로그램은 결국 "값에 이름표를 붙이고, 그 이름표로 값을 꺼내 쓰는 일"의 반복입니다.

---

## 1) 변수 (variable) — 값 한 개에 이름표 붙이기

```javascript
const userName = "철수"        // userName이라는 이름표를 "철수"에 붙임
const age = 30
let score = 0                  // 나중에 바뀔 값은 let
score = 100                    // 이렇게 바꿀 수 있음
```

- `const`: 한 번 정하면 못 바꾸는 값 (대부분 이걸 씀)
- `let`: 바뀔 수 있는 값
- `var`: 옛날 방식, AI가 만들면 `const`/`let`으로 바꿔달라고 하세요

Python에서는 그냥 `user_name = "철수"` 한 줄.

## 2) 배열 (array, list) — 여러 값을 순서대로 줄 세우기

```javascript
const fruits = ["사과", "바나나", "포도"]
fruits[0]      // "사과"  ← 0번부터 셈! 1번부터가 아님
fruits.length  // 3
fruits.push("딸기")   // 맨 뒤에 추가
```

Python에서는 `fruits = ["사과", "바나나", "포도"]`. 거의 똑같습니다.

**기억할 것:** 첫 번째 = `0`번. 0부터 센다는 게 처음엔 어색합니다.

## 3) 객체 (object, dict) — 이름표 여러 개를 한 묶음으로

```javascript
const user = {
  name: "철수",
  age: 30,
  email: "chulsoo@example.com"
}

user.name         // "철수"
user.age          // 30
user["email"]     // "chulsoo@example.com"  ← 이렇게도 꺼냄
```

Python에서는 `dict`라고 부릅니다:
```python
user = {"name": "철수", "age": 30}
user["name"]  # "철수"
```

**언제 배열, 언제 객체?**
- "사과, 바나나, 포도"처럼 같은 종류가 여러 개 → **배열**
- "이름은 철수, 나이는 30"처럼 항목마다 의미가 다름 → **객체**
- "사용자 100명" → 객체들의 배열: `[{name:"철수"}, {name:"영희"}, ...]`

## AI에게 적용

| 상황 | 이렇게 말하세요 |
|---|---|
| 같은 값(예: 색상 코드 `#3366ff`)이 코드 여러 곳에 반복됨 | "이 색상 값을 변수로 빼서 한 곳에서 관리하게 해줘" |
| `x`, `data`, `temp` 같은 이름이 보임 | "변수명을 의미가 드러나게 바꿔줘 — 예: `data`는 `userList`로" |
| AI가 매번 사용자 정보를 따로따로 주고받음 | "사용자 정보를 객체 하나로 묶어서 다뤄줘" |
| 데이터가 많아지면 한 줄씩 다루기 힘듦 | "배열로 만들고 반복문으로 처리해줘" |

**검수 포인트:** AI가 만든 코드에 설명 없는 숫자(`if (x > 86400)` 같은)가 보이면 → "그 86400이 뭔데?"라고 물어보고, 변수로 빼라고 요청. (정답: 86400은 하루의 초)

1. 다음 코드에서 변수, 배열, 객체를 각각 짚어보세요.
   ```javascript
   const todos = [
     { id: 1, text: "운동", done: false },
     { id: 2, text: "독서", done: true }
   ]
   const total = todos.length
   ```
2. `todos[1].text`의 값은?

<details>
<summary>정답</summary>

1. 변수: `todos`, `total`. 배열: `todos`. 객체: 배열 안의 `{id:1,...}`, `{id:2,...}` 두 개.
2. `"독서"` (1번 인덱스 = 두 번째 요소)
</details>
