# 01. 데이터를 담는 그릇 🍬

## 한 줄로 말하면

**값에 이름표 붙이기**예요. 그 이름으로 나중에 꺼내 써요.

코딩의 80%는 사실 이거예요. 진짜로요.

---

## 🤔 왜 필요한데요?

친구한테 "그거 좀 줘" 하면 "어떤 거?" 하잖아요.
"사과 줘" 하면 바로 가져다 줘요.

컴퓨터도 똑같아요. **이름이 있어야** 알아들어요.

---

## 1) 변수 — 이름표 한 개

```javascript
const myName = "철수"        // myName이라는 이름표를 "철수"에 붙임
const age = 30
let score = 0
```

이제 `myName`이라고 쓰면 컴퓨터는 "아, 철수 말하는 거구나" 합니다.

### 🤷 const? let? 뭐가 다름?

쉽게 외워요:

| 키워드 | 언제 써요 | 예시 |
|---|---|---|
| **const** | 안 바뀌는 것 | 생일, 학번, 이름 |
| **let** | 바뀔 수 있는 것 | 게임 점수, 나이 |
| ~~var~~ | 옛날 방식 | 안 써도 됨 |

```javascript
const birthday = "5월 5일"   // 절대 안 바뀜
let gameScore = 0            // 게임하면서 바뀜
gameScore = 100              // 이렇게 바꿀 수 있어요
```

> 💡 **`const`로 시작하세요.** 진짜 바꿔야 할 때만 `let`. 좋은 습관!

---

## 2) 배열 — 줄 세우기 🚂

```javascript
const fruits = ["사과", "바나나", "포도"]
```

**사과 → 바나나 → 포도** 이 순서로 줄 서있는 거예요.

번호로 꺼내요:

```javascript
fruits[0]      // "사과"
fruits[1]      // "바나나"
fruits[2]      // "포도"
fruits.length  // 3
```

### ⚠️ 0번부터 셉니다!

이게 진짜진짜 헷갈리는 부분이에요.

| 사람 | 컴퓨터 |
|---|---|
| 첫 번째 = 1번째 | 첫 번째 = **0번** |
| 두 번째 = 2번째 | 두 번째 = **1번** |
| 세 번째 = 3번째 | 세 번째 = **2번** |

`fruits[1]`이 "바나나"인 이유예요. **0부터!**

### 추가하기

```javascript
fruits.push("딸기")
// 이제 fruits = ["사과", "바나나", "포도", "딸기"]
```

`push`는 "맨 뒤에 끼워넣어"라는 뜻이에요.

---

## 3) 객체 — 명함 만들기 🎴

배열은 같은 종류 여러 개. 그러면 **한 사람의 여러 정보**는?

```javascript
const friend = {
  name: "철수",
  age: 12,
  email: "chulsoo@example.com"
}
```

이건 친구의 **명함**이에요. 점(`.`)으로 꺼내요:

```javascript
friend.name       // "철수"
friend.age        // 12
friend.email      // "chulsoo@example.com"
```

---

## 🍱 배열 vs 객체 — 어떤 걸 쓸까?

| 상황 | 답 |
|---|---|
| 사과, 바나나, 포도 (같은 종류 여러 개) | **배열** |
| 이름은 철수, 나이는 12 (한 사람 여러 정보) | **객체** |
| 친구 100명의 정보 | **배열 안에 객체들** ⭐ |

마지막 게 진짜 자주 봐요:

```javascript
const myFriends = [
  { name: "철수", age: 12 },
  { name: "영희", age: 11 },
  { name: "민수", age: 13 }
]

myFriends[0].name    // "철수"
myFriends[1].age     // 11
```

명함 여러 장이 줄 서있는 거예요.

## 🤖 AI한테 부탁할 때

| 이런 상황 | 이렇게 말해보세요 |
|---|---|
| 같은 색상 코드(`#3366ff`)가 여기저기 반복 | "이 색상을 변수로 빼서 한 곳에서 관리하게 해줘" |
| `x`, `data`, `temp` 같은 모호한 이름 | "변수명을 의미 있게 바꿔줘. 예: `data` → `userList`" |
| 사용자 정보를 매번 따로따로 다룸 | "사용자 정보를 객체 하나로 묶어줘" |

### 💡 검수 꿀팁

AI 코드에 **설명 없는 숫자**가 보이면 의심하세요!

```javascript
if (timer > 86400) { ... }   // ← 86400이 뭔데??
```

"그 86400이 뭔데?" 하고 물어보세요. **변수로 빼라**고 하면:

```javascript
const SECONDS_IN_DAY = 86400
if (timer > SECONDS_IN_DAY) { ... }   // ← 아 하루구나!
```

(86400 = 하루 = 24시간 × 60분 × 60초)

```javascript
  { id: 2, text: "독서", done: true }
]
const total = todos.length
```

1. 변수, 배열, 객체를 각각 짚어보세요.
2. `todos[1].text`는 뭐가 나올까요?

<details>
<summary>👀 정답 보기</summary>

1. **변수**: `todos`, `total` / **배열**: `todos` 자체 / **객체**: 배열 안의 `{id:1,...}`, `{id:2,...}` 두 개
2. **`"독서"`** — 1번 인덱스니까 **두 번째**! (0부터 셈)
