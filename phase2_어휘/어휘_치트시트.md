# 어휘 치트시트

vibe coding 중에 모르는 단어가 나오면 여기서 빠르게 확인. Ctrl+F로 검색.

---

## 데이터를 담는 것

| 단어 | 한 줄 설명 |
|---|---|
| **변수 (variable)** | 값에 붙은 이름표. `const x = 5` |
| **상수 (constant)** | 안 바뀌는 값. JavaScript에서는 `const` |
| **배열 (array, list)** | 순서 있는 값 목록. `[1, 2, 3]` |
| **객체 (object, dict)** | 이름표 묶음. `{name: "철수", age: 30}` |
| **null** | "의도적으로 비어있음" |
| **undefined** | "값이 정의되지 않음" |
| **NaN** | "Not a Number" — 숫자 계산 망함 |

## 동작을 묶는 것

| 단어 | 한 줄 설명 |
|---|---|
| **함수 (function)** | 재료 → 결과를 뱉는 기계 |
| **인자 (parameter, argument)** | 함수에 넣는 재료 |
| **반환값 (return value)** | 함수가 뱉는 결과 |
| **호출 (call)** | 함수를 실제 실행 |
| **메서드 (method)** | 객체/클래스에 붙은 함수 |
| **콜백 (callback)** | 다른 함수에 넘겨서 나중에 실행되게 하는 함수 |
| **화살표 함수 (arrow function)** | `(x) => x * 2` 형태 |

## 흐름 제어

| 단어 | 한 줄 설명 |
|---|---|
| **조건문 (if/else)** | 갈림길 |
| **반복문 (for/while)** | 같은 동작 반복 |
| **map** | 배열의 각 항목을 변환 |
| **filter** | 배열에서 조건 맞는 것만 남김 |
| **reduce** | 배열을 하나의 값으로 합침 |
| **break / continue** | 반복문 중단 / 다음 반복으로 |

## 타입

| 단어 | 한 줄 설명 |
|---|---|
| **string** | 문자열, "안녕" |
| **number** | 숫자, 30 |
| **boolean** | true/false |
| **타입 (type)** | 값의 종류 |
| **타입 강제 (type coercion)** | JS가 알아서 타입 바꾸는 것 (자주 버그) |
| **TypeScript** | 타입을 명시하는 JS |

## 모듈/라이브러리

| 단어 | 한 줄 설명 |
|---|---|
| **import / export** | 다른 파일에서 가져오기 / 내보내기 |
| **모듈 (module)** | 한 파일 단위 |
| **라이브러리 (library)** | 남이 만든 코드 묶음 |
| **의존성 (dependency)** | 우리 프로젝트가 쓰는 라이브러리 |
| **npm / pip** | JavaScript / Python의 패키지 관리자 |
| **package.json** | npm 의존성 명세서 |
| **node_modules** | 다운로드된 라이브러리 폴더 |
| **package-lock.json** | 정확한 버전 잠금 파일 |

## 네트워크/API

| 단어 | 한 줄 설명 |
|---|---|
| **API** | 다른 프로그램에 말 거는 창구 |
| **HTTP** | 웹의 통신 규약 |
| **GET/POST/PUT/DELETE** | 가져오기/만들기/수정/삭제 |
| **요청 (request)** | 우리가 보내는 것 |
| **응답 (response)** | 서버가 돌려주는 것 |
| **상태 코드 (status code)** | 200 성공, 400 우리잘못, 500 서버잘못 |
| **헤더 (headers)** | 메타정보 (인증 등) |
| **본문 (body)** | 실제 데이터 |
| **JSON** | 데이터 표준 형식 |
| **endpoint** | API의 한 URL |
| **REST** | 가장 흔한 API 스타일 |
| **CORS** | 도메인 다른 곳 호출 시 만나는 보안 규칙 |

## 비동기

| 단어 | 한 줄 설명 |
|---|---|
| **동기 (sync)** | 한 줄씩 차례로 실행 |
| **비동기 (async)** | 오래 걸리는 일을 기다리지 않고 진행 |
| **Promise** | 나중에 끝날 작업을 나타내는 객체 |
| **async/await** | Promise를 쉽게 쓰는 문법 |
| **then / catch** | Promise의 옛날 처리 방식 |

## 객체지향

| 단어 | 한 줄 설명 |
|---|---|
| **클래스 (class)** | 설계도 |
| **인스턴스 (instance)** | 설계도로 찍어낸 실제 |
| **constructor** | 인스턴스 만들 때 실행되는 함수 |
| **this / self** | "지금 이 인스턴스" |
| **상속 (inheritance)** | 다른 클래스의 기능 물려받기 |
| **new** | 인스턴스 생성 키워드 |

## 보안/환경

| 단어 | 한 줄 설명 |
|---|---|
| **환경변수 (env variable)** | 코드 밖에 두는 설정값 |
| **.env** | 환경변수를 적는 파일 |
| **시크릿 (secret)** | API 키, 비밀번호 등 노출 금지 값 |
| **.gitignore** | git에 안 올릴 파일 목록 |
| **인증 (authentication)** | "너 누구야?" |
| **권한 (authorization)** | "그거 해도 돼?" |

## 도구/환경

| 단어 | 한 줄 설명 |
|---|---|
| **터미널/CLI** | 명령어 입력하는 창 |
| **에디터 (editor)** | 코드 쓰는 프로그램 (VS Code, Cursor) |
| **컴파일 (compile)** | 사람 코드 → 기계 코드로 변환 |
| **번들 (bundle)** | 여러 파일을 하나로 합치는 빌드 결과 |
| **린트 (lint)** | 코드 스타일/문법 자동 검사 |
| **포맷 (format)** | 들여쓰기/줄바꿈 자동 정리 |
| **테스트 (test)** | 코드가 의도대로 동작하는지 확인하는 코드 |
| **디버그 (debug)** | 버그 찾고 고치기 |

## React/프론트엔드 자주 보는 단어

| 단어 | 한 줄 설명 |
|---|---|
| **컴포넌트 (component)** | UI의 한 조각 (버튼, 카드 등) |
| **props** | 컴포넌트에 전달하는 인자 |
| **state** | 컴포넌트 내부의 값, 바뀌면 화면 다시 그려짐 |
| **hook** | useState, useEffect 같은 React 내장 함수 |
| **렌더 (render)** | 컴포넌트를 화면에 그리는 것 |
| **JSX** | HTML처럼 생긴 JavaScript 문법 |
