# 02. 동작을 묶는 단위 — 함수, 인자, 반환값

## 핵심 한 줄

함수 = "재료를 넣으면 결과를 뱉는 기계". 같은 동작을 여러 번 쓸 때 한 번만 정의해두고 호출합니다.

---

## 1) 함수 (function)

```javascript
function add(a, b) {        // a, b가 "재료"
  return a + b              // 결과를 "뱉는다"
}

const result = add(3, 5)    // result = 8
```

- `add` = 함수의 이름
- `(a, b)` = **인자(parameter, argument)** — 함수가 받는 재료
- `return ...` = **반환값(return value)** — 함수가 뱉는 결과
- `add(3, 5)` = **호출(call)** — 함수를 실제로 실행

Python:
```python
def add(a, b):
    return a + b

result = add(3, 5)
```

## 2) 인자가 없거나, 반환값이 없는 함수

```javascript
function sayHello() {              // 인자 없음
  console.log("안녕")              // 반환값 없음 (그냥 출력만)
}
sayHello()
```

## 3) 화살표 함수 (arrow function) — 짧게 쓰는 법

```javascript
const add = (a, b) => a + b
const double = x => x * 2
```

JavaScript에서 자주 봅니다. 같은 함수입니다, 모양만 다릅니다.

## 4) 함수 안에서 함수 부르기

```javascript
function double(x) { return x * 2 }
function quadruple(x) { return double(double(x)) }
quadruple(3)  // 12
```

큰 작업을 작은 함수들로 쪼개는 게 좋은 코드의 핵심입니다.

## AI에게 적용

| 상황 | 이렇게 말하세요 |
|---|---|
| 한 함수가 200줄 넘게 길어짐 | "이 함수를 더 작은 함수 3~4개로 쪼개줘. 각 함수는 한 가지 일만 하게" |
| 비슷한 코드가 두세 군데 반복됨 | "이 반복되는 부분을 함수로 빼서 재사용하게 해줘" |
| 함수가 뭘 하는지 이름만 봐서 모름 | "함수명을 동사로, 무슨 일을 하는지 드러나게 바꿔줘 (예: `process` → `convertCsvToJson`)" |
| AI가 만든 함수가 이상하게 동작함 | "이 함수에 `add(3, 5)`를 넣으면 뭐가 나와야 해?"라고 물어서 본인이 기대한 결과와 비교 |

**중요한 어휘 구분:**
- "함수를 **호출한다**(call)" = 실행한다
- "함수를 **정의한다**(define)" = 만든다
- "이 함수가 **반환한다**(returns)" = 결과로 이런 값을 뱉는다

이 단어들을 정확히 쓰면 AI가 의도를 더 잘 파악합니다.
