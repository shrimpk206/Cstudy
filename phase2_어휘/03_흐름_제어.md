# 03. 흐름 제어 — 조건문, 반복문

## 핵심 한 줄

코드는 기본적으로 위에서 아래로 한 줄씩 실행됩니다. 그 흐름을 **분기**시키거나 **반복**시키는 것이 if문/for문입니다.

---

## 1) 조건문 (if / else)

```javascript
const age = 20

if (age >= 19) {
  console.log("성인")
} else if (age >= 13) {
  console.log("청소년")
} else {
  console.log("어린이")
}
```

- `if (조건) { ... }` — 조건이 참이면 실행
- `else if` — 위 조건이 거짓이고, 이 조건이 참이면
- `else` — 위 조건들이 다 거짓이면

**비교 연산자:**
- `===` 같다 (JavaScript에서는 `==` 말고 `===`을 쓰세요)
- `!==` 다르다
- `>`, `<`, `>=`, `<=` 크다, 작다, 크거나 같다, 작거나 같다
- `&&` 그리고 (AND)
- `||` 또는 (OR)
- `!` 부정 (NOT)

```javascript
if (age >= 19 && hasId) { ... }     // 19살 이상이고 신분증 있으면
if (role === "admin" || isOwner) {  // 관리자거나 주인이면
```

## 2) 반복문 (for / while)

### `for` — 정해진 횟수만큼

```javascript
for (let i = 0; i < 5; i++) {
  console.log(i)
}
// 0, 1, 2, 3, 4 출력
```

세 부분: `시작; 조건; 매번 끝나고 할 일`. 처음엔 외계어처럼 보이지만 99% 위 형태입니다.

### `for...of` — 배열 한 개씩 꺼내기 (이게 더 자주 쓰임)

```javascript
const fruits = ["사과", "바나나", "포도"]
for (const fruit of fruits) {
  console.log(fruit)
}
```

### `while` — 조건이 참인 동안

```javascript
let count = 0
while (count < 3) {
  console.log(count)
  count++       // count = count + 1
}
```

### 배열 메서드 — 실전에서 가장 자주 보는 형태

```javascript
const numbers = [1, 2, 3, 4]

numbers.forEach(n => console.log(n))         // 하나씩 처리
const doubled = numbers.map(n => n * 2)      // [2, 4, 6, 8]
const evens = numbers.filter(n => n % 2 === 0)  // [2, 4]
const sum = numbers.reduce((a, b) => a + b, 0)  // 10
```

이 4개(`forEach`, `map`, `filter`, `reduce`)는 vibe coding 코드에서 정말 자주 봅니다.
- **map**: 각 항목을 변환
- **filter**: 조건에 맞는 항목만 남김
- **reduce**: 다 합쳐서 한 값으로
- **forEach**: 각 항목으로 뭔가 함 (반환 안 함)

## AI에게 적용

| 상황 | 이렇게 말하세요 |
|---|---|
| 조건이 너무 깊게 중첩됨 (`if`안에 `if`안에 `if`) | "이 if문 깊이를 줄이고, 가능하면 early return으로 바꿔줘" |
| `for`문이 복잡하게 꼬여 있음 | "이 반복문을 `map`/`filter`로 다시 써줄 수 있어?" |
| AI가 만든 조건이 의도와 다름 | "이 조건이 만족되는 경우를 예시로 3개 들어줘"라고 검증 |
| 무한 루프(영원히 안 끝남)에 빠짐 | "이 while문이 언제 종료되지? `count`가 어떻게 바뀌어야 끝나?" |

**조심:** AI가 `for (let i = 0; i < arr.length; i++)`로 만들었는데 그 안에서 `arr.push(...)`를 하면 끝나지 않습니다. 이런 미묘한 버그는 본인이 잡아야 할 때가 있습니다.
