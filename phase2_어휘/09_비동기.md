# 09. 비동기 ☕

## 한 줄로 말하면

**카페 진동벨**이에요.
주문하고 진동벨 받고, 음료 나올 때까지 다른 일 해도 되는 그거.

이 개념 모르면 vibe coding 코드의 절반은 안 돌아가요.

---

## ☕ 카페 비유

스타벅스에서 음료 주문하면:

1. 주문 ✋
2. 진동벨 받음 📳 ← 이게 **Promise**
3. 자리에 앉아서 핸드폰 봄 📱 ← 그동안 다른 일!
4. 진동벨 울림 🎉 ← Promise resolved
5. 음료 받음 ☕

만약 주문하고 음료 나올 때까지 그 자리에 서 있으면? 줄이 막혀버리죠.

---

## 1) Promise — "나중에 줄게" 약속

```javascript
const promise = fetch("https://api.example.com/users")
// promise = "응답 나중에 올 거야" 약속 (= 진동벨)
```

Promise는 세 상태 중 하나:

| 상태 | 의미 | 비유 |
|---|---|---|
| **pending** | 진행 중 | 음료 만드는 중 |
| **fulfilled** | 성공! 값 옴 | 진동벨 울림 ✅ |
| **rejected** | 실패! 에러 | "재료 없대요" 😢 |

---

## 2) async / await — Promise 쉽게 다루기

```javascript
async function getUser() {
  const response = await fetch("https://api.example.com/users/1")
  const data = await response.json()
  return data
}
```

해석:
- **`async`** = "이 함수는 비동기야!"
- **`await`** = "여기서 진동벨 울릴 때까지 기다려"

> 🔑 **`await`는 `async` 함수 안에서만** 쓸 수 있어요!

---

## 3) 호출하는 쪽도 await

```javascript
async function main() {
  const user = await getUser()
  console.log(user.name)
}
main()
```

`getUser()` 자체도 Promise를 돌려주니까 `await`로 받아야 해요.

### 🐛 가장 흔한 버그: await 빼먹기

```javascript
const data = fetch("...")    // ❌ Promise 객체 자체를 받음
const data = await fetch(...) // ✅ 응답 객체 받음
```

`console.log`했는데 `[object Promise]` 찍히면 99% **await 빼먹은 것!**

---

## 4) 에러 처리 — try / catch 🛡️

비동기는 자주 실패해요 (네트워크 끊김, 서버 다운).

```javascript
async function getUser() {
  try {
    const response = await fetch("https://api.example.com/users/1")
    if (!response.ok) throw new Error("응답 실패: " + response.status)
    return await response.json()
  } catch (error) {
    console.error("뭔가 잘못됨:", error)
    return null
  }
}
```

`try { ... } catch (error) { ... }`로 감싸세요. 안 그러면 한 번 실패에 앱 통째로 깨져요.

## 5) 😱 함정 3대장

### 함정 1: forEach 안의 await는 안 먹혀요!!

```javascript
// ❌ 이거 작동 안 해요
users.forEach(async user => {
  await save(user)
})
console.log("저장 완료")  // 저장 끝나기 전에 찍힘!

// ✅ for...of 사용
for (const user of users) {
  await save(user)
}
console.log("저장 완료")  // 진짜 다 끝나고 찍힘
```

### 함정 2: 직렬 vs 병렬 — Promise.all 활용

```javascript
// ❌ 한 번에 하나씩 (느림 — 순서대로 9초)
const a = await fetch(url1)  // 3초
const b = await fetch(url2)  // 3초
const c = await fetch(url3)  // 3초

// ✅ 동시에! (빠름 — 3초)
const [a, b, c] = await Promise.all([
  fetch(url1),
  fetch(url2),
  fetch(url3),
])
```

> 💡 독립적인 호출은 **Promise.all**로 한꺼번에! 같은 시간에 더 많이.

### 함정 3: try/catch 빼먹기

```javascript
// ❌ 실패하면 앱 깨짐
const data = await fetch("/risky")

// ✅ 안전망
try {
  const data = await fetch("/risky")
} catch (e) {
  // 처리
}
```

## 6) Python의 비동기

```python
import asyncio

async def get_user():
    await asyncio.sleep(1)
    return {"name": "철수"}

asyncio.run(get_user())
```

개념 같아요. 단, JavaScript처럼 흔하지 않아요. 보통 `requests` 같은 동기로 충분.

| 이런 상황 | 이렇게 말해보세요 |
|---|---|
| API 결과가 `[object Promise]` 찍힘 | "여기 await 빼먹은 듯. async/await 적용" |
| 여러 API 호출이 너무 느림 | "독립적이면 Promise.all로 병렬 처리" |
| `Uncaught (in promise) Error` | "async 함수에 try/catch 추가" |
| 콜백 지옥(.then().then()) | "async/await로 다시 써줘" |

### 💡 검수 꿀팁

Promise 반환하는 함수(주로 fetch, axios, 데이터베이스 호출) 앞에 **`await` 빠진 곳**이 있나 보세요. 99% 버그입니다.
