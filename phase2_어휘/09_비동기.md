# 09. 비동기 — async, await, Promise

## 핵심 한 줄

"오래 걸리는 일(API 호출, 파일 읽기)이 끝날 때까지 기다리는 법". 이걸 이해 못 하면 vibe coding 코드의 절반은 안 돌아갑니다.

---

## 1) 왜 비동기가 필요한가

API 호출은 0.5초쯤 걸립니다. 그동안 나머지 코드를 멈춰 세울까요? 그러면 화면이 멈춰버립니다.

JavaScript는 그래서 "끝나면 알려줘"라는 방식을 씁니다. 이게 비동기.

## 2) Promise — "나중에 끝날 약속"

```javascript
const promise = fetch("https://api.example.com/users")
// promise 변수는 "응답이 나중에 올 것"을 나타내는 객체
```

Promise는 세 상태 중 하나:
- **pending**: 아직 진행 중
- **fulfilled**: 성공해서 값이 나옴
- **rejected**: 실패해서 에러가 남

## 3) async / await — Promise를 쉽게 다루는 법

```javascript
async function getUser() {
  const response = await fetch("https://api.example.com/users/1")
  const data = await response.json()
  return data
}
```

- `async`: 이 함수는 비동기다 (Promise를 반환함)
- `await`: 이 줄에서 Promise가 끝날 때까지 기다린다

규칙: **`await`는 `async` 함수 안에서만** 쓸 수 있습니다 (예외: 최신 환경의 top-level).

## 4) 호출하는 쪽도 await

```javascript
async function main() {
  const user = await getUser()
  console.log(user.name)
}
main()
```

`getUser()` 자체도 Promise를 반환하므로 `await`로 받아야 합니다.

## 5) 에러 처리 — try/catch

```javascript
async function getUser() {
  try {
    const response = await fetch("https://api.example.com/users/1")
    if (!response.ok) throw new Error("응답 실패: " + response.status)
    return await response.json()
  } catch (error) {
    console.error("뭔가 잘못됨:", error)
    return null
  }
}
```

비동기 작업은 자주 실패합니다 (네트워크 끊김, 서버 다운 등). `try/catch`로 감싸세요.

## 6) 자주 만나는 함정

### 함정 1: `await` 빼먹기
```javascript
const data = fetch("...")    // ❌ Promise 객체 자체를 받음 (값 아님)
const data = await fetch(...) // ✅ 응답 객체를 받음
```

### 함정 2: `forEach` 안의 await는 안 먹힘
```javascript
// ❌ 작동 안 함
users.forEach(async user => {
  await save(user)
})

// ✅ for...of 사용
for (const user of users) {
  await save(user)
}
```

### 함정 3: 병렬 처리 — `Promise.all`
```javascript
// ❌ 한 번에 하나씩 — 느림
const a = await fetch(url1)
const b = await fetch(url2)
const c = await fetch(url3)

// ✅ 동시에 — 빠름
const [a, b, c] = await Promise.all([
  fetch(url1),
  fetch(url2),
  fetch(url3),
])
```

## 7) Python의 비동기

```python
import asyncio

async def get_user():
    await asyncio.sleep(1)
    return {"name": "철수"}

asyncio.run(get_user())
```

개념 동일. JavaScript만큼 흔하진 않습니다. 보통은 `requests` 같은 동기 라이브러리로 충분.

## AI에게 적용

| 상황 | 이렇게 말하세요 |
|---|---|
| API 결과가 `[object Promise]`로 찍힘 | "여기 await 빼먹은 거 같아. async/await 적용해줘" |
| 여러 API 호출이 너무 느림 | "이 호출들 독립적이면 Promise.all로 병렬 처리해줘" |
| `Uncaught (in promise) Error...` 에러 | "async 함수에 try/catch 추가해서 에러 처리해줘" |
| 콜백 지옥(중첩된 .then().then())이 보임 | "async/await로 다시 써줘" |

**검수 포인트:**
- `await`가 빠진 곳이 있는지 보세요. Promise를 반환하는 함수(주로 fetch, axios, 데이터베이스 호출)에 `await`가 없으면 99% 버그.
- `try/catch`로 감싸지 않은 `await`는 실패 시 앱이 통째로 깨집니다.

   ```javascript
   function getUser() {
     return fetch("/api/user").json()
   }
   ```
2. 10개의 URL을 동시에 호출하려면?

<summary>정답</summary>

1. `fetch`는 Promise를 반환하므로 `.json()`을 그 Promise에 직접 호출할 수 없음. `await`나 `.then()`으로 풀어야 함:
   ```javascript
   async function getUser() {
     const res = await fetch("/api/user")
     return await res.json()
   }
   ```
