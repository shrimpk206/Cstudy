# 04. 타입과 널 — string, number, boolean, null, undefined

## 핵심 한 줄

값에는 **종류(타입)**가 있습니다. "30"(글자)과 30(숫자)은 다릅니다. 이 차이가 vibe coding 버그의 절반입니다.

---

## 1) 기본 타입 5종

| 타입 | 예시 | 한국어 |
|---|---|---|
| **string** | `"안녕"`, `"30"`, `'a'` | 글자(문자열) — 따옴표로 감쌈 |
| **number** | `30`, `3.14`, `-5` | 숫자 — 따옴표 없음 |
| **boolean** | `true`, `false` | 참/거짓 |
| **null** | `null` | "값이 의도적으로 비어있다" |
| **undefined** | `undefined` | "값이 아예 정의되지 않았다" |

```javascript
const name = "철수"      // string
const age = 30           // number
const isAdult = true     // boolean
const car = null         // 일부러 비워둠
let score                // 값을 안 줘서 → undefined
```

## 2) "30"과 30은 다르다

```javascript
"30" + 1     // "301"  ← 글자에 1을 붙임
30 + 1       // 31     ← 숫자 더하기
"30" * 1     // 30     ← 곱하기는 자동 변환됨 (헷갈림 주의)

"30" === 30  // false  ← 타입까지 같아야 ===
```

특히 폼 입력값은 항상 string으로 들어옵니다. 숫자로 쓰려면 변환:
```javascript
const ageStr = "30"          // 폼에서 받은 값
const age = Number(ageStr)   // 30 (number)
const age2 = parseInt(ageStr) // 30 (number, 정수만)
```

## 3) null vs undefined — 둘 다 "값이 없다"인데 왜 두 개?

- **undefined**: 시스템이 "이건 아직 값이 없네"라고 자동으로 표시한 것
- **null**: 개발자가 의도적으로 "이건 비워둘게"라고 적은 것

실전에서는 거의 같은 의미라고 봐도 됩니다. 단, **둘 다 에러를 자주 일으킵니다**:

```javascript
const user = null
user.name      // ❌ TypeError: Cannot read properties of null
```

## 4) 타입 검사

```javascript
typeof "안녕"     // "string"
typeof 30         // "number"
typeof true       // "boolean"
typeof null       // "object"  ← 함정! 옛날 버그가 굳어진 것
typeof undefined  // "undefined"

Array.isArray([1,2])  // true   ← 배열인지 확인
```

## 5) Python의 타입

거의 같습니다. 이름만 조금 다름:
- `str`, `int`, `float`, `bool`
- 빈 값: `None` (JavaScript의 null/undefined를 합친 느낌)

## 6) TypeScript — 타입을 미리 적는 JavaScript

```typescript
const name: string = "철수"
const age: number = 30

function add(a: number, b: number): number {
  return a + b
}
```

vibe coding에서 점점 자주 보입니다. JavaScript에 `:type`이 추가됐을 뿐입니다.

## AI에게 적용

| 상황 | 이렇게 말하세요 |
|---|---|
| 숫자 계산이 이상함 (예: `"3" + "5"` = `"35"`) | "이 값이 string인지 number인지 확인해줘. 폼 입력이면 변환 추가" |
| `Cannot read property X of undefined` 에러 | "이 객체가 undefined일 가능성을 처리해줘 (옵셔널 체이닝 `?.` 또는 if 검사)" |
| AI가 너무 자유분방하게 코드를 짠다 | "TypeScript로 작성하고 함수마다 타입을 명시해줘" |
| 계산 결과에 `NaN` (Not a Number)이 나옴 | "어디서 숫자가 아닌 값이 섞였는지 찾아줘. `Number()` 변환이 빠진 곳 있을 듯" |

## 자가점검

1. 다음 결과는?
   ```javascript
   "10" + 5
   "10" - 5
   typeof null
   ```
2. 어떤 함수가 가끔 `undefined`를 반환합니다. 호출한 쪽에서 `result.name`을 했더니 에러. 어떻게 막을까?

<details>
<summary>정답</summary>

1. `"105"`, `5`, `"object"` — `+`는 글자 붙이기 우선, `-`는 강제로 숫자 변환, `null`의 typeof는 "object"(역사적 버그).
2. `if (result) { result.name }` 또는 `result?.name` (옵셔널 체이닝).
</details>
