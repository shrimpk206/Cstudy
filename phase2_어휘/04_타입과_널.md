# 04. 타입과 널 🍭

## 한 줄로 말하면

값에는 **종류(타입)**가 있어요.
**`"30"`(글자 사탕)과 `30`(숫자 사탕)은 다른 거예요.**

이거 모르면 진짜 황당한 버그 만나요.

---

## 🍬 사탕 가게 비유

가게에 두 종류 사탕이 있어요:

- **🍬 글자 사탕**: `"안녕"`, `"30"`, `"a"` ← 따옴표로 싸여있어요
- **🔢 숫자 사탕**: `30`, `3.14`, `-5` ← 따옴표 없음

겉으로 똑같아 보여도 **완전히 다른 사탕**이에요.

---

## 1) 5가지 기본 타입

| 타입 | 예 | 한국어 |
|---|---|---|
| **string** | `"안녕"`, `"30"` | 글자 (따옴표) |
| **number** | `30`, `3.14` | 숫자 |
| **boolean** | `true`, `false` | 참/거짓 |
| **null** | `null` | 일부러 비움 |
| **undefined** | `undefined` | 아직 안 정함 |

```javascript
const name = "철수"      // string
const age = 30           // number
const isAdult = true     // boolean
const car = null         // 일부러 비움
let score                // 안 줘서 → undefined
```

---

## 2) 😱 헷갈리는 함정

### `"30" + 1`은 뭐가 나올까요?

```javascript
"30" + 1     // ???
```

답은 **`"301"`** 이에요.

😱 왜요? `+` 연산자가 한쪽이 글자(string)면 **글자 붙이기**로 작동하거든요.
"30"이라는 글자에 "1"을 붙이니까 "301"!

### 더 헷갈리는 거

```javascript
"30" + 1     // "301"  ← 붙이기
"30" - 1     // 29     ← 빼기는 자동으로 숫자!
"30" * 1     // 30     ← 곱하기도
"30" === 30  // false  ← 글자 30과 숫자 30은 달라요
```

뺄셈, 곱셈은 알아서 숫자로 바꿔주고, 덧셈만 글자 붙이기 우선이에요. **JavaScript의 가장 큰 함정!**

### 폼 입력은 항상 글자!

```javascript
const ageStr = "30"               // 폼에서 받은 값 (글자!)
const age = Number(ageStr)        // 30 (이제 숫자)
const age2 = parseInt(ageStr)     // 30 (이것도 숫자)
```

> 💡 **폼/API에서 받은 값은 일단 글자라고 의심하세요.** 계산 전에 `Number()`로 변환!

---

## 3) null vs undefined — 둘 다 "비어있다" 🤔

| 비교 | 의미 | 비유 |
|---|---|---|
| **null** | 일부러 비웠어 | 사탕 통을 일부러 비워둠 |
| **undefined** | 아직 안 정해짐 | 사탕 통이 아예 없음 |

실전에서는 거의 같이 다뤄도 돼요. 단 둘 다 **에러를 자주** 일으켜요:

```javascript
const user = null
user.name      // ❌ TypeError: Cannot read properties of null
```

null/undefined인 애한테서 뭘 꺼내려고 하면 컴퓨터가 화내요.

---

## 4) 옵셔널 체이닝 `?.` — 안전 장치 🛡️

```javascript
const user = null

user.name      // ❌ 에러! 앱 깨짐!
user?.name     // ✅ undefined (에러 안 남)
```

`?.`는 "있으면 꺼내고, 없으면 그냥 undefined 줘"라는 뜻이에요.

API에서 값 받아올 때 진짜 자주 써요:

```javascript
const response = await fetch("/user")
const data = await response.json()
console.log(data.user?.profile?.name)  // 중간에 비어도 안 깨짐
```

## 5) 타입 검사

```javascript
typeof "안녕"     // "string"
typeof 30         // "number"
typeof true       // "boolean"
typeof undefined  // "undefined"
typeof null       // "object"  ← 함정! 옛날 버그가 굳어진 것

Array.isArray([1,2])  // true   ← 배열인지 확인
```

> 🐛 `typeof null`이 `"object"`인 건 JavaScript의 유명한 버그예요. 25년 전부터 그래요. 그냥 외워요.

---

## 6) Python의 타입

거의 같아요. 이름만 조금 다름:

| JavaScript | Python |
|---|---|
| string | `str` |
| number | `int`, `float` |
| boolean | `bool` |
| null/undefined | `None` |

---

## 7) TypeScript — 타입을 미리 적기

```typescript
const name: string = "철수"
const age: number = 30

function add(a: number, b: number): number {
  return a + b
}
```

JavaScript에 `: 타입`만 추가됐어요. **타입을 잘못 쓰면 미리 잡아주는 안전벨트**같은 거예요. 요즘 vibe coding 코드에 점점 자주 보여요.

## 🤖 AI한테 부탁할 때

| 이런 상황 | 이렇게 말해보세요 |
|---|---|
| 숫자 계산이 이상함 (`"3"+"5"` = `"35"`) | "이 값이 string인지 number인지 확인. 폼 입력이면 변환 추가" |
| `Cannot read property X of undefined` 에러 | "이 객체가 undefined일 가능성 처리해줘. 옵셔널 체이닝 `?.`" |
| AI 코드가 너무 자유분방 | "TypeScript로, 함수마다 타입 명시해줘" |
| `NaN`이 결과에 나옴 | "어디서 숫자가 아닌 값이 섞였는지 찾아줘. `Number()` 빠진 곳" |

### 💡 검수 꿀팁

AI가 외부 데이터(API 응답, 폼 입력, JSON)를 받자마자 바로 쓰면 의심하세요.
항상 두 질문 던져요:

1. "**비어있을 수 있나?**" → null 가드 또는 `?.`
2. "**타입이 string인가 number인가?**" → 변환 필요?
