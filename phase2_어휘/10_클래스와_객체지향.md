# 10. 클래스와 객체지향

## 핵심 한 줄

같은 종류의 데이터+동작을 묶은 "설계도"가 클래스. 그 설계도로 찍어낸 게 인스턴스. vibe coding에서 직접 만들 일은 적지만, **읽을 줄은 알아야 합니다**.

---

## 1) 클래스 (class)

```javascript
class User {
  constructor(name, email) {
    this.name = name
    this.email = email
  }

  greet() {
    return "안녕, 나는 " + this.name
  }
}

const u = new User("철수", "c@example.com")  // 인스턴스 만들기
u.greet()                                      // "안녕, 나는 철수"
```

- **class**: 설계도
- **constructor**: 인스턴스를 만들 때 처음 실행되는 함수
- **this**: "지금 만들어지는 이 인스턴스"
- **new**: "이 설계도로 새 인스턴스 만들어"

## 2) 메서드 (method)

클래스 안에 정의된 함수 = 메서드. 위 예에서 `greet()`이 메서드.

배열에서 본 `arr.push(...)`, `arr.map(...)`도 다 Array 클래스의 메서드입니다.

## 3) 상속 (inheritance)

```javascript
class Admin extends User {
  constructor(name, email, role) {
    super(name, email)   // 부모 클래스의 constructor 호출
    this.role = role
  }

  delete(target) {
    console.log(this.name + "이 " + target + "을 삭제")
  }
}
```

`Admin`은 `User`의 모든 기능 + 자기 추가 기능. **상속을 너무 깊게 쓰면 코드 복잡해집니다**. AI도 종종 과하게 씁니다.

## 4) 언제 클래스를 쓰나

vibe coding에서 클래스가 필요한 경우는 의외로 적습니다:

- ✅ **상태를 가진 컴포넌트** (옛날 React)
- ✅ **연결된 자원** (데이터베이스 연결, 게임 캐릭터)
- ✅ **외부 라이브러리가 클래스를 제공할 때** (예: `new Date()`)
- ❌ 단순한 데이터 묶음 → 객체로 충분
- ❌ 한 번 쓰는 함수 → 그냥 함수

요즘 JavaScript는 **함수형 스타일**이 더 흔합니다. 객체 + 함수 조합으로 처리.

## 5) Python의 클래스

```python
class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email

    def greet(self):
        return f"안녕, 나는 {self.name}"

u = User("철수", "c@example.com")
u.greet()
```

`__init__`이 constructor, `self`가 this. `new` 없이 `User(...)`만 적습니다.

## 6) 자주 보는 클래스들

이미 표준에 깔린 클래스를 사용하는 패턴:
```javascript
new Date()                          // 현재 시각
new Map()                           // 키-값 저장소 (객체보다 강력)
new Set([1, 2, 3, 2])               // 중복 없는 집합 → {1,2,3}
new URL("https://example.com")      // URL 파싱
new Error("뭔가 잘못됨")             // 에러 객체
```

`new`가 보이면 "클래스의 인스턴스를 만드는 중이구나"라고 읽으세요.

## AI에게 적용

| 상황 | 이렇게 말하세요 |
|---|---|
| AI가 단순한 일에 클래스를 5개 만들어 옴 | "이거 함수만으로 충분할 거 같은데, 클래스 없이 다시 짜줘" |
| 클래스 상속이 3단계 이상 깊음 | "상속 계층이 너무 깊어. 평탄하게(composition으로) 다시 설계해줘" |
| `this`가 `undefined`라고 에러 남 | "여기 화살표 함수 vs 일반 함수 문제야. this 바인딩 점검해줘" |
| 클래스 코드가 100줄 넘게 길어짐 | "이 클래스 책임이 너무 많아. 두 개로 쪼갤 수 있어?" |

**읽기 팁:**
- `class X extends Y` → "X는 Y의 일종이고, Y의 모든 걸 물려받는다"
- `this.xxx` → "이 인스턴스의 속성 xxx"
- `super(...)` → "부모 클래스의 같은 이름 메서드 호출"
