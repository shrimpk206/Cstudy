# 10. 클래스와 객체지향 🍪

## 한 줄로 말하면

**쿠키 틀**이에요.
틀로 똑같은 모양 쿠키를 여러 개 찍어내는 거.

vibe coding에서 직접 만들 일은 적지만, **읽을 줄은 알아야** 해요.

---

## 🍪 쿠키 비유

쿠키 가게에서:

- **쿠키 틀** = 클래스 (설계도)
- **틀로 찍어낸 쿠키 한 개** = 인스턴스 (실제)
- **틀로 같은 모양 100개** = 인스턴스 100개

같은 모양 쿠키가 100개 필요하면 매번 손으로 깎지 말고 **틀 하나로** 찍어내요!

---

## 1) 클래스 만들기

```javascript
class User {
  constructor(name, email) {
    this.name = name
    this.email = email
  }

  greet() {
    return "안녕, 나는 " + this.name
  }
}

const u = new User("철수", "c@example.com")  // 쿠키 한 개 찍기!
u.greet()                                      // "안녕, 나는 철수"
```

용어:

| 용어 | 비유 | 설명 |
|---|---|---|
| **class** | 쿠키 틀 | 설계도 |
| **constructor** | 틀에 반죽 넣기 | 인스턴스 만들 때 처음 실행 |
| **this** | "이 쿠키" | 지금 만들어지는 인스턴스 |
| **new** | "찍어내!" | 인스턴스 생성 |

---

## 2) 메서드 — 쿠키의 능력

클래스 안에 정의된 함수가 **메서드**예요. 위 예에서 `greet()`.

배열의 `arr.push(...)`, `arr.map(...)`도 사실 다 Array 클래스의 메서드!

---

## 3) 상속 — 쿠키 진화하기 🧬

```javascript
class Admin extends User {
  constructor(name, email, role) {
    super(name, email)   // 부모(User)의 constructor 호출
    this.role = role
  }

  delete(target) {
    console.log(this.name + "이 " + target + "을 삭제")
  }
}
```

`Admin`은 `User`의 **모든 능력 + 자기만의 추가 능력**.

> ⚠️ 상속을 너무 깊게 쓰면 코드 복잡해져요. AI도 종종 과하게 씁니다.

---

## 4) 언제 클래스를 쓰나요?

vibe coding에서 클래스가 꼭 필요한 경우는 **의외로 적어요**:

✅ 쓸 만한 경우:
- 상태를 가진 자원 (데이터베이스 연결, 게임 캐릭터)
- 외부 라이브러리가 클래스를 제공할 때 (`new Date()`)
- 옛날 React 컴포넌트

❌ 쓸 필요 없는 경우:
- 단순 데이터 묶음 → **객체로 충분**
- 한 번 쓰는 함수 → 그냥 함수
- AI가 그냥 "제대로" 해보겠다고 만든 클래스

요즘 JavaScript는 **함수형 스타일**이 더 흔해요.

---

## 5) Python의 클래스

```python
class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email

    def greet(self):
        return f"안녕, 나는 {self.name}"

u = User("철수", "c@example.com")
u.greet()
```

`__init__`이 constructor, `self`가 this. **`new` 없이** `User(...)`만!

---

## 6) 자주 보는 클래스 — `new`가 보이면 인스턴스!

```javascript
new Date()                          // 현재 시각
new Map()                           // 키-값 저장소
new Set([1, 2, 3, 2])               // 중복 없는 집합 → {1,2,3}
new URL("https://example.com")      // URL 파싱
new Error("뭔가 잘못됨")             // 에러 객체
```

코드 읽다가 `new`가 보이면 **"클래스 인스턴스 만드는 중!"** 하고 알아챌 수 있어요.

## 🤖 AI한테 부탁할 때

| 이런 상황 | 이렇게 말해보세요 |
|---|---|
| AI가 단순한 일에 클래스 5개 만듦 | "함수만으로 충분할 듯, 클래스 없이 다시" |
| 상속이 3단계 이상 깊음 | "상속 너무 깊어. 평탄하게(composition) 다시" |
| `this`가 `undefined`라고 에러 | "화살표 함수 vs 일반 함수 문제. this 바인딩 점검" |
| 클래스가 100줄 넘게 큼 | "이 클래스 책임 너무 많아. 두 개로 쪼개기" |

### 💡 코드 읽기 팁

- `class X extends Y` → "X는 Y의 일종이고 Y의 모든 걸 물려받음"
- `this.xxx` → "이 인스턴스의 속성 xxx"
- `super(...)` → "부모 클래스의 같은 이름 메서드 호출"
